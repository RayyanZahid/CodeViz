  <action>
    Create shared type definitions following the exact patterns from 01-RESEARCH.md:

    1. `packages/shared/src/types/graph.ts`:
       - `NodeType` const object with values: `service_module`, `component_page`, `data_store`, `external_api`
       - `NodeType` type (derived from const)
       - `EdgeType` const object with values: `imports_depends_on`, `calls_invokes`, `reads_writes`, `publishes_subscribes`
       - `EdgeType` type (derived from const)
       - `GraphNode` interface: id (string), name (string), nodeType (NodeType), zone (string | null), fileList (string[]), incomingEdgeCount (number), outgoingEdgeCount (number), lastModified (Date)
       - `GraphEdge` interface: id (string), sourceId (string), targetId (string), edgeType (EdgeType)

    2. `packages/shared/src/types/events.ts`:
       - `ChangeEventType` const object with values: `node_added`, `node_removed`, `node_updated`, `edge_added`, `edge_removed`, `zone_changed`
       - `ChangeEventType` type (derived from const)
       - `ChangeEventPayload` discriminated union type (discriminant: `type` field matching ChangeEventType values)
       - `ChangeEvent` interface: id (number, sequence), eventType (ChangeEventType), payload (ChangeEventPayload), timestamp (Date)

    3. `packages/shared/src/types/messages.ts`:
       - Import `GraphNode`, `GraphEdge` from `./graph.js`
       - `GraphDeltaMessage` interface: type: 'graph_delta', version (number), addedNodes (GraphNode[]), removedNodeIds (string[]), updatedNodes (GraphNode[]), addedEdges (GraphEdge[]), removedEdgeIds (string[])
       - `InitialStateMessage` interface: type: 'initial_state', version (number), nodes (GraphNode[]), edges (GraphEdge[]), layoutPositions (Record<string, { x: number; y: number; zone: string | null }>)
       - `ServerMessage` = GraphDeltaMessage | InitialStateMessage

    4. `packages/shared/src/types/index.ts` — barrel re-export: `export * from './graph.js'`, `export * from './events.js'`, `export * from './messages.js'`

    NOTE on import extensions: Use `.js` extensions in all imports within shared and server packages (NodeNext requires this). Client package (Bundler resolution) does not require extensions but use them for consistency.

    5. `packages/client/vite.config.ts`:
       - Import defineConfig from 'vite'
       - server.proxy: `/ws` → target `http://localhost:3100`, ws: true, changeOrigin: false
       - server.proxy: `/api` → target `http://localhost:3100`, changeOrigin: false

    6. `packages/client/index.html` — minimal HTML5 boilerplate with `<div id="app"></div>` and `<script type="module" src="/src/main.ts"></script>`

    7. `packages/client/src/main.ts` — placeholder: `document.getElementById('app')!.innerHTML = '<h1>ArchLens</h1>'`
  </action>
