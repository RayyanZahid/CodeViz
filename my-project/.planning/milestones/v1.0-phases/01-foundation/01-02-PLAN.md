<task type="auto">
  <name>Task 1: Create SQLite connection with WAL mode and Drizzle schema for all 4 tables</name>
  <files>
    packages/server/drizzle.config.ts
    packages/server/src/db/connection.ts
    packages/server/src/db/schema.ts
  </files>
  <action>
    1. `packages/server/src/db/connection.ts`:
       - Import Database from 'better-sqlite3' and drizzle from 'drizzle-orm/better-sqlite3'
       - Import * as schema from './schema.js'
       - Create `new Database('./archlens.db')` — file path relative to where server runs
       - Set `sqlite.pragma('journal_mode = WAL')` immediately after opening
       - Set `sqlite.pragma('synchronous = NORMAL')` for better WAL write performance
       - Export `const db = drizzle(sqlite, { schema })`

    2. `packages/server/src/db/schema.ts`:
       - Import `sqliteTable, text, integer, real` from 'drizzle-orm/sqlite-core'

       **graphNodes table:**
       - id: text primary key
       - name: text not null
       - nodeType: text not null (values: 'service_module' | 'component_page' | 'data_store' | 'external_api')
       - zone: text nullable (set in Phase 4)
       - fileList: text mode json, $type<string[]>, not null, default []
       - incomingEdgeCount: integer not null default 0
       - outgoingEdgeCount: integer not null default 0
       - lastModified: integer mode timestamp_ms not null
       - createdAt: integer mode timestamp_ms not null

       **graphEdges table:**
       - id: text primary key
       - sourceId: text not null, references graphNodes.id
       - targetId: text not null, references graphNodes.id
       - edgeType: text not null (values: 'imports_depends_on' | 'calls_invokes' | 'reads_writes' | 'publishes_subscribes')
       - createdAt: integer mode timestamp_ms not null

       **changeEvents table (append-only):**
       - id: integer primary key autoIncrement (sequence number)
       - eventType: text not null (values from ChangeEventType)
       - payload: text mode json, $type<Record<string, unknown>>, not null
       - timestamp: integer mode timestamp_ms not null

       **layoutPositions table:**
       - nodeId: text primary key, references graphNodes.id
       - x: real not null default 0
       - y: real not null default 0
       - zone: text nullable
       - updatedAt: integer mode timestamp_ms not null

    3. `packages/server/drizzle.config.ts`:
       - Import defineConfig from 'drizzle-kit'
       - dialect: 'sqlite', schema: './src/db/schema.ts', out: './drizzle', dbCredentials: { url: './archlens.db' }

    After creating these files, run `cd packages/server && npx drizzle-kit push` to create the database with all tables. Verify WAL mode: run a quick script `node -e "import Database from 'better-sqlite3'; const db = new Database('./archlens.db'); console.log(db.pragma('journal_mode'))"` (from packages/server directory, or use tsx).
  </action>
  <verify>
    Run `cd packages/server && npx drizzle-kit push` — exits 0, creates archlens.db with all 4 tables.
    Run `cd packages/server && npx tsx -e "import Database from 'better-sqlite3'; const db = new Database('./archlens.db'); console.log(db.pragma('journal_mode'))"` — outputs `[{ journal_mode: 'wal' }]`.

  </verify>

<task type="auto">
  <name>Task 2: Create repository layer with typed CRUD operations for all entities</name>
  <files>
    packages/server/src/db/repository/nodes.ts
    packages/server/src/db/repository/edges.ts
    packages/server/src/db/repository/events.ts
    packages/server/src/db/repository/positions.ts

  <action>
    Create one repository file per domain entity. Each repository imports `db` from `../connection.js`, the relevant table from `../schema.js`, and `eq` (plus other operators as needed) from `drizzle-orm`.

    1. `packages/server/src/db/repository/nodes.ts` — export `nodesRepository` object:
       - `findById(id: string)` — select from graphNodes where id matches, return first or undefined
       - `findAll()` — select all from graphNodes
       - `upsert(node)` — insert with onConflictDoUpdate on id, updating name, nodeType, zone, fileList, incomingEdgeCount, outgoingEdgeCount, lastModified
       - `deleteById(id: string)` — delete from graphNodes where id matches
       - Use `InferSelectModel` and `InferInsertModel` from drizzle-orm for type safety

    2. `packages/server/src/db/repository/edges.ts` — export `edgesRepository` object:
       - `findById(id: string)` — select where id matches
       - `findAll()` — select all
       - `findByNodeId(nodeId: string)` — select where sourceId or targetId matches (use `or()` from drizzle-orm)
       - `insert(edge)` — insert new edge
       - `deleteById(id: string)` — delete where id matches
       - `deleteByNodeId(nodeId: string)` — delete all edges connected to a node

    3. `packages/server/src/db/repository/events.ts` — export `eventsRepository` object:
       - `append(event)` — insert only (no update, no delete — append-only table)
       - `findAll()` — select all ordered by id ascending (sequence order)
       - `findSince(sequenceId: number)` — select where id > sequenceId, ordered ascending
       - `getLatestSequence()` — select max(id) from changeEvents, return number or 0 if empty

    4. `packages/server/src/db/repository/positions.ts` — export `positionsRepository` object:
       - `findByNodeId(nodeId: string)` — select where nodeId matches
       - `findAll()` — select all (used for initial state on client connect)
       - `upsert(position)` — insert with onConflictDoUpdate on nodeId, updating x, y, zone, updatedAt
       - `deleteByNodeId(nodeId: string)` — delete where nodeId matches

    All repository methods use synchronous Drizzle API (better-sqlite3 is synchronous). Return types should use the inferred Drizzle model types. Use `.js` extensions on all import paths (NodeNext ESM requirement).
  </action>
