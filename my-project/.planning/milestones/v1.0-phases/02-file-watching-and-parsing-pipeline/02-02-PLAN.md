<objective>
Implement the tree-sitter parsing worker pool with TypeScript/JavaScript grammar support, incremental parse tree caching, and extraction of imports, exports, and call relationships.

Purpose: This is the core parsing engine. Worker threads parse source files using tree-sitter native bindings without blocking the main event loop. Incremental parsing reuses previous trees for faster re-parses. Extractors turn ASTs into plain-object ParseResults.
Output: ParserPool class, compiled worker file, TypeScript and JavaScript extractors.
</objective>

  <action>
    **1. Create `packages/server/src/parser/extractors/typescript.ts`:**

    Export function `extractTypeScript(rootNode: Parser.SyntaxNode, filePath: string): { imports: ImportInfo[], exports: ExportInfo[], calls: CallInfo[] }`

    **Import extraction:**
    - Walk `rootNode.namedChildren` looking for `import_statement` nodes
    - For each import_statement, find the `string` child (the source specifier), strip quotes
    - Detect type-only imports: check if the import has `type` keyword (node type `type_identifier` or `import_statement` with `"type"` text before the clause)
    - A simpler approach: check if `node.text.startsWith('import type')` for type-only detection
    - Return array of `ImportInfo { source, isTypeOnly }`

    **Export extraction:**
    - Walk for `export_statement` nodes
    - Extract exported name from the declaration child (function_declaration.name, class_declaration.name, variable_declarator.name, or `'default'` for default exports)
    - Detect type-only: `node.text.startsWith('export type')`
    - For `export { ... }` re-exports, extract each identifier from the export_clause
    - Return array of `ExportInfo { name, isDefault, isTypeOnly }`

    **Call extraction (per Claude's Discretion — direct calls only for v1):**
    - Recursively walk all nodes looking for `call_expression`
    - Extract callee: if the `function` child is an `identifier`, use its text (direct call like `createServer()`)
    - Skip if callee is `member_expression` (method chains like `app.listen()` — too noisy for v1)
    - Determine `isTopLevel`: true if the call_expression's parent chain reaches program/module without entering a function/method body
    - Return array of `CallInfo { callee, isTopLevel }`

    Use `import type Parser from 'tree-sitter';` for the SyntaxNode type. Import `ImportInfo`, `ExportInfo`, `CallInfo` from `@archlens/shared`.

    **2. Create `packages/server/src/parser/extractors/javascript.ts`:**

    Export function `extractJavaScript(rootNode: Parser.SyntaxNode, filePath: string): { imports: ImportInfo[], exports: ExportInfo[], calls: CallInfo[] }`

    JavaScript extraction is very similar to TypeScript but simpler:
    - Same `import_statement` node type for imports
    - Same `export_statement` for exports
    - No type-only imports/exports — always set `isTypeOnly: false`
    - Same `call_expression` handling for calls

    To avoid duplication, the JavaScript extractor can delegate to the TypeScript extractor and simply set all `isTypeOnly` flags to `false`. Or share a common `extractBase` helper. Choose the cleaner approach.

    **3. Create `packages/server/src/parser/worker.ts`:**

    This is the piscina worker file (will be compiled to JS in Task 2).

    **Grammar cache (module-level, initialized once per worker thread):**
    ```typescript
    import Parser from 'tree-sitter';
    import { typescript, tsx } from 'tree-sitter-typescript';
    import JavaScript from 'tree-sitter-javascript';
    ```
    Create a `Map<SupportedLanguage, Parser>` with pre-initialized parsers at module top-level. Each parser calls `new Parser()` and `setLanguage()` once.

    **Tree cache for incremental parsing (PARSE-06):**
    - `const treeCache = new Map<string, Parser.Tree>();`
    - Worker-local cache (trees cannot cross thread boundaries — confirmed in research)
    - On parse: if `treeCache.has(filePath)`, pass old tree as second arg to `parser.parse(source, oldTree)` for incremental reuse
    - After parse: update `treeCache.set(filePath, newTree)`
    - Use pass-old-tree strategy (no tree.edit()) per research recommendation — good enough for file-level changes

    **Default export (piscina handler):**
    ```typescript
    export default function parseFileTask(task: ParseTask): ParseResult { ... }
    ```
    - Select parser by `task.language`
    - Read tree cache, parse (full or incremental)
    - Select extractor by language (typescript extractor for ts/tsx, javascript for js/jsx)
    - Extract imports, exports, calls
    - Update tree cache
    - Check `tree.rootNode.hasError` for `hasErrors` flag
    - Measure parse time with `performance.now()`
    - Return plain ParseResult object (PARSE-05: tree goes out of scope after extraction; held only in cache for incremental reuse)

    **Named export for cache eviction:**
    ```typescript
    export function evictFile(filePath: string): void {
      treeCache.delete(filePath);
    }
    ```

    Import types from `@archlens/shared` using `.js` extension. Import extractors from `./extractors/typescript.js` and `./extractors/javascript.js`.

    **2. Create `packages/server/src/parser/ParserPool.ts`:**

    **Class: ParserPool**
    - Constructor takes optional `options: { minThreads?: number, maxThreads?: number, idleTimeout?: number }`
    - Creates `Piscina` instance with:
      - `filename`: path to compiled worker — `new URL('../../dist/parser/worker.js', import.meta.url).href`
      - `minThreads: options.minThreads ?? 1`
      - `maxThreads: options.maxThreads ?? Math.max(2, Math.floor(os.availableParallelism() / 2))`
      - `idleTimeout: options.idleTimeout ?? 30_000`

    **Methods:**
    - `async parseFile(task: ParseTask): Promise<ParseResult>` — calls `this.pool.run(task)` and returns result
    - `async parseBatch(tasks: ParseTask[]): Promise<ParseResult[]>` — runs `Promise.all(tasks.map(t => this.parseFile(t)))` for parallel parsing within a batch
    - `async evictFile(filePath: string): Promise<void>` — calls `this.pool.run(filePath, { name: 'evictFile' })` to evict tree cache in all workers (piscina named task support)
    - `async destroy(): Promise<void>` — calls `this.pool.destroy()` for graceful shutdown
    - `get stats(): { completed: number, queued: number }` — returns pool utilization from `this.pool.completed` and `this.pool.queueSize`

    Import `Piscina` from `'piscina'`, `os` from `'node:os'`, types from `@archlens/shared`.

    **CRITICAL:** The worker file must exist at `dist/parser/worker.js` at runtime. The `build:workers` script handles this. If the file doesn't exist when ParserPool is constructed, throw a clear error: "Worker file not found at dist/parser/worker.js. Run 'pnpm build:workers' first."

    Add a pre-check in the constructor:
    ```typescript
    import { existsSync } from 'node:fs';
    import { fileURLToPath } from 'node:url';
    const workerPath = new URL('../../dist/parser/worker.js', import.meta.url);
    if (!existsSync(fileURLToPath(workerPath))) {
      throw new Error("Worker file not found. Run 'pnpm build:workers' first.");
    }
    ```
