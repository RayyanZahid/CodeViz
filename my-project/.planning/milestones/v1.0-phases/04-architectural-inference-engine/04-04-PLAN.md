  <name>Task 1: Build InferenceEngine orchestrator class</name>
  <files>packages/server/src/inference/InferenceEngine.ts</files>
  <action>
Create `packages/server/src/inference/InferenceEngine.ts` — the central orchestrator that subscribes to DependencyGraph delta events and coordinates all inference modules.

**Typed EventEmitter pattern** (matches DependencyGraph pattern from Phase 3):
```typescript
import { EventEmitter } from 'node:events';

interface InferenceEngineEvents {
  inference: [result: InferenceResult];
}

export class InferenceEngine extends EventEmitter<InferenceEngineEvents> {
```

**Constructor(graph: DependencyGraph, watchRoot: string):**
1. Create `configLoader = new ConfigLoader(watchRoot)`.
2. Create `classifier = new ZoneClassifier(configLoader, graph)`.
3. Create `corroborator = new EventCorroborator()`.
4. Create `riskDetector = new RiskDetector()`.
5. Subscribe to graph delta events: `graph.on('delta', (delta) => this.processDelta(delta))`.
6. Store `graph` reference for zone lookups.

**processDelta(delta: GraphDelta): void** — the core processing pipeline:
1. **Zone classification:** `const zoneUpdates = this.classifier.classifyDelta(delta)`.
2. **Persist zone updates:** For each zoneUpdate, update the zone column on the corresponding graph_nodes row and append a zone_changed event to changeEvents:
   ```typescript
   for (const update of zoneUpdates) {
     // Update zone column on existing node row
     db.update(graphNodes).set({ zone: update.zone }).where(eq(graphNodes.id, update.nodeId)).run();
     // Append zone_changed event
     eventsRepository.append({
       eventType: ChangeEventType.ZONE_CHANGED,
       payload: { type: 'zone_changed', nodeId: update.nodeId, oldZone: update.previousZone, newZone: update.zone },
       timestamp: new Date(),
     });
   }
   ```
   Import `db` from `../db/connection.js`, `graphNodes` from `../db/schema.js`, `eq` from `drizzle-orm`, `eventsRepository` from `../db/repository/events.js`, and `ChangeEventType` from `@archlens/shared/types`.

3. **Event corroboration:** `const architecturalEvents = this.corroborator.processDelta(delta, zoneUpdates)`.

4. **Risk detection:** `const risks = this.riskDetector.detectRisks(delta, (id) => this.getZoneForNode(id), (id) => this.graph.getOutEdges(id))`.

5. **Emit inference result** (only if there's something to report):
   ```typescript
   if (zoneUpdates.length > 0 || architecturalEvents.length > 0 || risks.length > 0) {
     this.emit('inference', { zoneUpdates, architecturalEvents, risks, graphVersion: delta.version });
   }
   ```

**getZoneForNode(nodeId: string): ZoneName** — helper that returns the zone from the classifier's cache, falling back to 'unknown'.

**loadPersistedZones(): void** — called at startup to seed the ZoneClassifier cache from SQLite:
1. Load all node rows from `nodesRepository.findAll()`.
2. For each row with a non-null zone, call `this.classifier.updateZoneCache(row.id, row.zone as ZoneName)`.
This ensures that after a server restart, the classifier doesn't re-classify everything as 'unknown' and then gradually recover — it starts with the correct zones from the last session.

**destroy(): void** — calls `this.configLoader.destroy()` for graceful shutdown.

**Key design decisions (from RESEARCH.md):**
- Zone classification, corroboration, and risk detection are all synchronous and cheap (O(edges) worst case). No async needed. Matches DependencyGraph's synchronous delta processing pattern.
- Zone updates to SQLite are fire-and-forget (no transaction coupling with Phase 3's persistence). Zone is nullable — a missing zone on restart is recoverable via re-classification.
- The InferenceEngine does NOT own graph structure writes — only zone column updates and changeEvent logging.

**Add inference event logging** (after the existing graph delta listener):
```typescript
inferenceEngine.on('inference', (result) => {
  const parts: string[] = [];
  if (result.zoneUpdates.length > 0) parts.push(`${result.zoneUpdates.length} zone updates`);
  if (result.architecturalEvents.length > 0) parts.push(`${result.architecturalEvents.length} arch events`);
  if (result.risks.length > 0) parts.push(`${result.risks.length} risks`);
  console.log(`[Inference] v${result.graphVersion}: ${parts.join(', ')}`);
});
```

**Update the onClose hook** to also destroy the inference engine:
```typescript
fastify.addHook('onClose', async () => {
  inferenceEngine.destroy();
  await pipeline.stop();
});
```

**Ordering in index.ts must be:**
1. `const graph = new DependencyGraph();`
2. `graph.loadFromDatabase();`
3. `const watchRoot = process.env.ARCHLENS_WATCH_ROOT ?? process.cwd();`
4. `const inferenceEngine = new InferenceEngine(graph, watchRoot);`
5. `inferenceEngine.loadPersistedZones();`
6. Graph delta logger (existing)
7. Inference event logger (new)
8. `const pipeline = new Pipeline(watchRoot, ...);`
9. onClose hook (updated)
10. `start()` function

This order ensures:
- InferenceEngine subscribes to graph.on('delta') BEFORE the Pipeline starts emitting events.
- loadPersistedZones seeds the classifier cache with SQLite zones BEFORE chokidar initial scan.
- The delta event flows: Pipeline -> DependencyGraph -> InferenceEngine -> console log.
