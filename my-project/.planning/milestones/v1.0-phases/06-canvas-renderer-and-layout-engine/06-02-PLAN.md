1. **Create canvas/NodeRenderer.ts** — imperative class managing Konva Groups (Rect + Text) on the graph layer:

   - Import `Konva` from `konva` (default import for imperative API)
   - Import `{ getZoneLayout }` from `../layout/ZoneConfig.js`
   - Import `type { GraphNode }` from `@archlens/shared/types`

   - Define `Position = { x: number; y: number }` type
   - Define `NODE_BASE_SIZE = 100` and `NODE_HEIGHT_RATIO = 0.6` constants
   - Define `MIN_NODE_SIZE = 70` and `MAX_NODE_SIZE = 160` for scaling range

   - Class `NodeRenderer`:
     - Private `layer: Konva.Layer` (passed in constructor)
     - Private `shapes: Map<string, Konva.Group>` — tracks created groups by node ID
     - Private `positions: Map<string, Position>` — current positions for each node

     - Method `setPosition(nodeId: string, pos: Position)`:
       - Store in positions map
       - If shape exists, call shape.x(pos.x) and shape.y(pos.y)

     - Method `getPosition(nodeId: string): Position | undefined`:
       - Return from positions map

     - Method `syncAll(nodes: Map<string, GraphNode>)`:
       - For each node in the map:
         - If shape exists: update fill color, label, size if changed
         - If shape does not exist: create new Group with Rect + Text
       - For each shape not in nodes map: destroy and remove from shapes map
       - Call `layer.batchDraw()`
       - Node creation:
         - Compute size: `clamp(NODE_BASE_SIZE + (node.incomingEdgeCount + node.outgoingEdgeCount) * 4, MIN_NODE_SIZE, MAX_NODE_SIZE)` (per CONTEXT.md: node size scaled by connection count)
         - Get zone color from `getZoneLayout(node.zone ?? 'unknown').fillColor`
         - Create Konva.Group with id = node.id
         - Add Konva.Rect: width=size, height=size*NODE_HEIGHT_RATIO, offsetX=size/2, offsetY=(size*NODE_HEIGHT_RATIO)/2, fill=zoneColor, stroke='#ffffff20', strokeWidth=1, cornerRadius=6, perfectDrawEnabled=false, shadowForStrokeEnabled=false
         - Add Konva.Text: text=node.name (truncate to 18 chars with ellipsis if longer), fontSize=11, fill='#ffffffdd', align='center', width=size-8, offsetX=(size-8)/2, offsetY=8, listening=false
         - Position group at stored position or (0,0) initially
         - Add group to layer

     - Method `applyDelta(currentNodes: Map<string, GraphNode>, prevNodes: Map<string, GraphNode>)`:
       - Added: nodes in current not in prev -> create shapes
       - Removed: nodes in prev not in current -> destroy shapes
       - Updated: nodes in both where zone or edge counts changed -> update fill/size

     - Method `getShape(nodeId: string): Konva.Group | undefined`:
       - Return from shapes map (used by AnimationQueue in Plan 04)

     - Method `getAllNodeBounds(): Array<{ id: string; x: number; y: number; width: number; height: number }>`:
       - Return array of bounding boxes for all nodes (used by CullingIndex)

     - Method `setVisible(nodeId: string, visible: boolean)`:
       - Get shape, call shape.visible(visible) — for viewport culling

2. **Create canvas/EdgeRenderer.ts** — imperative class managing Konva Lines on the graph layer:

   - Import `Konva` from `konva`
   - Import `type { GraphEdge }` from `@archlens/shared/types`
   - Import `type { NodeRenderer }` from `./NodeRenderer.js`

   - Class `EdgeRenderer`:
     - Private `layer: Konva.Layer`
     - Private `lines: Map<string, Konva.Line>` — tracks edge lines by edge ID
     - Private `nodeRenderer: NodeRenderer` (passed in constructor — reads node positions)

     - Method `syncAll(edges: Map<string, GraphEdge>)`:
       - For each edge:
         - Get source pos from nodeRenderer.getPosition(edge.sourceId)
         - Get target pos from nodeRenderer.getPosition(edge.targetId)
         - If either position missing, skip (node may not be placed yet)
         - If line exists: update points
         - If line does not exist: create Konva.Line
       - For lines not in edges map: destroy and remove
       - Call `layer.batchDraw()`
       - Line creation: points=[srcX, srcY, tgtX, tgtY], stroke='#ffffff18', strokeWidth=1, tension=0.3 (bezier per CONTEXT.md), listening=false, perfectDrawEnabled=false
       - Add small arrowhead: add a second Konva.Arrow or use Konva.Arrow instead of Konva.Line — Konva.Arrow has built-in pointerLength and pointerWidth. Use Konva.Arrow with pointerLength=8, pointerWidth=6, stroke='#ffffff18', fill='#ffffff18'

     - Method `applyDelta(currentEdges: Map<string, GraphEdge>, prevEdges: Map<string, GraphEdge>)`:
       - Added/removed edges: create or destroy arrows

     - Method `updatePositions()`:
       - Re-read all node positions from nodeRenderer and update arrow points
       - Called after layout engine runs (Plan 03)

     - Method `getLine(edgeId: string): Konva.Arrow | undefined`:
       - Return from lines map (used by AnimationQueue Plan 04 for edge glow)

     - Method `setVisible(edgeId: string, visible: boolean)`:
       - Get line, call line.visible(visible)

1. **Create canvas/ZoneRenderer.ts** — imperative class drawing zone background rectangles:

   - Import `Konva` from `konva`
   - Import `{ ZONE_LAYOUTS }` from `../layout/ZoneConfig.js`

   - Class `ZoneRenderer`:
     - Private `layer: Konva.Layer`
     - Private `zoneGroups: Konva.Group[]`

     - Constructor(layer: Konva.Layer):
       - Store layer
       - Call `renderZones()` immediately

     - Private `renderZones()`:
       - For each zone in ZONE_LAYOUTS:
         - Create Konva.Group({ listening: false })
         - Add Konva.Rect: x=zone.bounds.x0, y=zone.bounds.y0, width=(x1-x0), height=(y1-y0), fill=zone.bgColor, cornerRadius=8, listening=false
         - Add Konva.Text: x=zone.bounds.x0+12, y=zone.bounds.y0+10, text=zone.label, fontSize=14, fill='#ffffff25', fontStyle='bold', listening=false
         - Add group to layer
       - Store groups in zoneGroups
       - Move all zone groups to bottom of layer (zones behind nodes): `group.moveToBottom()`
       - `layer.batchDraw()`

     - Method `updateLabelVisibility(scale: number)`:
       - If scale > 0.8: hide zone labels (too zoomed in — labels compete with nodes)
       - If scale <= 0.8: show zone labels (zoomed out — labels help orientation)
       - Per CONTEXT.md: "Zone labels visible at low zoom levels, hidden when zoomed in"

2. **Create canvas/CullingIndex.ts** — quadtree-based viewport culling:

   - Import `Quadtree` from `@timohausmann/quadtree-js` (or import default — check package export)
   - Import `type { NodeRenderer }` from `./NodeRenderer.js`
   - Import `type { EdgeRenderer }` from `./EdgeRenderer.js`

   - Define `ViewportRect = { x: number; y: number; width: number; height: number }`

   - Class `CullingIndex`:
     - Private `quadtree: Quadtree`
     - Private `nodeRenderer: NodeRenderer`
     - Private `edgeRenderer: EdgeRenderer`
     - Private `lastVisibleNodeIds: Set<string>`

     - Constructor(nodeRenderer, edgeRenderer, canvasWidth: number, canvasHeight: number):
       - Initialize quadtree with canvas bounds: `new Quadtree({ x: 0, y: 0, width: canvasWidth, height: canvasHeight })`
       - Store references

     - Method `rebuild()`:
       - Clear quadtree
       - For each node bounds from nodeRenderer.getAllNodeBounds():
         - Insert into quadtree: `{ x, y, width, height, id }`
       - Called after layout engine places nodes (infrequent — only on new nodes)

     - Method `updateVisibility(viewport: ViewportRect)`:
       - Query quadtree: `quadtree.retrieve(viewport)` returns nodes overlapping viewport
       - Build set of visible node IDs
       - For each node:
         - If in visible set and not in lastVisibleNodeIds: nodeRenderer.setVisible(id, true)
         - If not in visible set and in lastVisibleNodeIds: nodeRenderer.setVisible(id, false)
       - For edges: an edge is visible if either its source or target is visible
         - Track visible edges similarly and toggle visibility
       - Update lastVisibleNodeIds
       - NOTE: At 300 nodes, the quadtree retrieve is O(log n). The full iterate for edge visibility is O(edges) but edges are lightweight (one Line shape).

     - Method `getVisibleNodeCount(): number`:
       - Return lastVisibleNodeIds.size (for performance debugging)

   NOTE: The viewport rectangle passed to updateVisibility must be in STAGE (world) coordinates, not screen coordinates. The ViewportController (Plan 04) will compute this as: `{ x: -stage.x() / scale, y: -stage.y() / scale, width: stage.width() / scale, height: stage.height() / scale }`.
