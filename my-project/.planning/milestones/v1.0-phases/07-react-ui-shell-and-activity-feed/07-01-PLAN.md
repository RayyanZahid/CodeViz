    **1. Create `packages/client/src/utils/eventSentence.ts`:**

    Pure function `toSentence(event: ArchitecturalEvent, nodeNameFn: (id: string) => string): string` that maps event types to terse technical sentences (per user decision: "terse technical tone").

    Import `ArchitecturalEventType` from `@archlens/shared/types`. Switch on `event.type`:
    - `COMPONENT_CREATED` → `"${name} created"`
    - `COMPONENT_SPLIT` → `"${name} split"`
    - `COMPONENT_MERGED` → `"${name} merged"`
    - `DEPENDENCY_ADDED` → `"${name} → depends on ${targetName}"` (use arrow per user example)
    - `DEPENDENCY_REMOVED` → `"${name} → removed dep on ${targetName}"`
    - default → `"${name} updated"`

    The `nodeNameFn` extracts a display name from a node ID (typically `graphStore.getState().nodes.get(id)?.name ?? id`). This keeps the function pure and testable.

    **2. Create `packages/client/src/store/inferenceStore.ts`:**

    Follow the exact same Zustand pattern as `graphStore.ts` — double-paren `create<InferenceStore>()((set, get) => ({...}))` with a vanilla reference export.

    **Types:**
    ```typescript
    interface ActivityItem {
      id: string;               // `${timestamp}-${event.nodeId}`
      event: ArchitecturalEvent;
      sentence: string;         // pre-computed via toSentence()
      timestamp: number;
      iconColor: string;        // green=#22c55e for creation, orange=#f97316 for risk, blue=#3b82f6 for dependency change
    }

    interface RiskItem {
      id: string;               // fingerprint: `${signal.type}:${sortedAffectedNodeIds.join(',')}`
      signal: RiskSignal;
      firstSeen: number;
      reviewed: boolean;
    }
    ```

    **State:**
    - `activityFeed: ActivityItem[]` — max 50, newest first
    - `risks: Map<string, RiskItem>` — keyed by fingerprint
    - `activeNodeIds: Map<string, number>` — nodeId → timestamp of last activity (for UI-03/UI-04 decay tracking)

    **Actions:**
    - `applyInference(msg: InferenceMessage): void` — the main entry point called by wsClient:
      1. For each `msg.architecturalEvents`: create `ActivityItem` with `toSentence()`, using `graphStore.getState().nodes.get(id)?.name ?? id` as the nodeNameFn. Assign `iconColor` based on event type (COMPONENT_CREATED → green, DEPENDENCY_ADDED/REMOVED → blue, COMPONENT_SPLIT/MERGED → blue). Post-hoc merge: if the last feed item has the same `event.nodeId` and occurred < 2 seconds ago, replace with summary `"N events for ${name}"` (per user decision: batch rapid changes into summaries).
      2. For `msg.risks`: compute fingerprint for each signal. If fingerprint exists in `risks` Map and is `reviewed: true`, preserve `reviewed` state but update `signal`/`details`. If fingerprint is new, add with `reviewed: false`. Remove any existing fingerprints NOT in the current risks array (risk resolved).
      3. For `msg.architecturalEvents` node IDs + any `msg.zoneUpdates` node IDs: update `activeNodeIds` Map with current timestamp.
      4. Prepend new activity items to `activityFeed`, then slice to max 50 items.
      5. Call `set({...})` with new state (immutable update — new Map copies for risks and activeNodeIds).

    - `markRiskReviewed(riskId: string): void` — copy risks Map, set `reviewed: true` for the given fingerprint, call `set({ risks })`.

    - `pruneExpiredActive(): void` — remove entries from `activeNodeIds` older than 30 seconds. Called on a setInterval(30000) managed outside the store (in App.tsx or wsClient).

    **Exports:**
    - `export const useInferenceStore = create<InferenceStore>()(...)` (React hook)
    - `export const inferenceStore = useInferenceStore` (vanilla reference for wsClient)

    **Risk fingerprint function** (internal to the module):
    ```typescript
    function riskFingerprint(signal: RiskSignal): string {
      if (signal.affectedNodeIds && signal.affectedNodeIds.length > 0) {
        return `${signal.type}:${[...signal.affectedNodeIds].sort().join(',')}`;
      }
      return `${signal.type}:${signal.nodeId}`;
    }
    ```

    **Icon color mapping** (internal helper):
    - `component_created` → `'#22c55e'` (green)
    - `dependency_added`, `dependency_removed` → `'#3b82f6'` (blue)
    - `component_split`, `component_merged` → `'#3b82f6'` (blue)
    - default → `'#94a3b8'` (gray)

    **3. Wire `packages/client/src/ws/wsClient.ts`:**

    Add import at top: `import { inferenceStore } from '../store/inferenceStore.js';` and `import type { InferenceMessage } from '@archlens/shared/types';`

    Replace the `case 'inference'` block (lines 196-199) from:
    ```typescript
    case 'inference': {
      console.log('[WS] Inference:', msg);
      break;
    }
    ```
    To:
    ```typescript
    case 'inference': {
      inferenceStore.getState().applyInference(msg as unknown as InferenceMessage);
      break;
    }
    ```

    This mirrors the existing pattern for `graphStore.getState().applyDelta(msg)`. The `as unknown as InferenceMessage` cast bridges Zod's relaxed string types to the strict shared types (same pattern used for graph_delta and initial_state cases).
