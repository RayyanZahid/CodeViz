  <action>
Review NodeInspector.tsx (as rewritten by Plan 01) and verify the dependency sections meet all CONTEXT.md requirements. Fix any gaps:

**Dependency aggregation verification:**
- Outgoing edges: filter `edges` where `edge.sourceId === selectedNodeId`, group by `edge.targetId`
- For each target group, compute total import count: sum `edge.dependencyCount ?? 1` across all edges to that target
- Display as: "ComponentName (N imports)" where N is the summed count
- Sort by count descending — heaviest dependencies at top
- Incoming edges: same pattern but filter where `edge.targetId === selectedNodeId`, group by `edge.sourceId`

**Count badge styling:**
- Count badge is a small inline element: `(N imports)` in muted text (`#ffffff55`, fontSize 10)
- Component name is primary color (`#ffffffcc`) and the count follows in lighter text
- If count is 1, display "(1 import)" (singular)

**Clickable navigation:**
- Each dependency row must have `cursor: pointer` and `onClick={() => onHighlightNode?.(nodeId)}`
- On hover, dependency row should show a subtle background highlight (`rgba(255,255,255,0.05)`)
- Per CONTEXT.md: "Dependency links should feel like navigating a wiki — click a dependency name, you're instantly looking at that component"
- The onHighlightNode callback in App.tsx already calls `canvasRef.current.selectNodeOnCanvas(nodeId)` which selects the node and updates inspector, then calls `viewportController.panToNode()` for smooth canvas panning
- Verify this chain works: clicking dep name → onHighlightNode → selectNodeOnCanvas (which calls handleSelectNodeRef.current(nodeId) → sets selectedNodeId) + panToNode → inspector swaps to new component + canvas centers on it

**Smooth pan animation (from CONTEXT.md specifics):**
- "Pan-to-component on dependency click should be a smooth animation, not a jump"
- Review ViewportController.panToNode() — if it uses Konva.Tween for smooth animation, no changes needed. If it jumps directly (sets stage position), wrap in a short tween: `new Konva.Tween({ node: stage, x: targetX, y: targetY, duration: 0.3, easing: Konva.Easings.EaseInOut }).play()`
- If ViewportController needs modification for smooth panning, update `packages/client/src/canvas/ViewportController.ts` — but ONLY if panToNode does a hard jump. Check first by reading the file.

**Edge cases:**
- Component with no outgoing deps: show "None" in muted text
- Component with no incoming deps: show "None" in muted text
- Self-referencing edge (sourceId === targetId): skip from both lists (component importing from itself is not meaningful to display)
- Dependency target that no longer exists in nodes Map: show the raw ID as fallback name
