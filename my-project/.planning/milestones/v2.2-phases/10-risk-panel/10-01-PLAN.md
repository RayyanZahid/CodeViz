<task type="auto">
  <name>Task 1: Add localStorage persistence and resurface logic to inferenceStore</name>
  <files>packages/client/src/store/inferenceStore.ts</files>
  <action>
Modify the inferenceStore to persist reviewed risk IDs in localStorage and resurface previously-reviewed risks when they re-fire.

**localStorage persistence:**
1. Define a constant `REVIEWED_RISKS_KEY = 'archlens-reviewed-risks'`.
2. Add a helper `loadReviewedRisks(): Set<string>` that reads from `localStorage.getItem(REVIEWED_RISKS_KEY)`, parses the JSON array of fingerprint strings, and returns a Set. Return empty Set on parse failure.
3. Add a helper `saveReviewedRisks(risks: Map<string, RiskItem>): void` that extracts all fingerprints where `reviewed === true`, serializes as JSON array string, and calls `localStorage.setItem(REVIEWED_RISKS_KEY, ...)`.
4. In the store's initial state, after creating the empty `risks` Map, load the reviewed set from localStorage so it's available for the first applyInference call. Store the loaded set in a module-level variable `let persistedReviewedIds: Set<string> = loadReviewedRisks()`.
5. In `markRiskReviewed`, after setting the risk to `reviewed: true` in the Map, call `saveReviewedRisks(newRisks)` to persist.
6. In `applyInference`, when processing incoming risks (section 3), for new risks (not already in the Map), check if the fingerprint is in `persistedReviewedIds`. If it IS, set `reviewed: true` for that risk. This hydrates prior reviewed state from localStorage.

**Resurface logic (per CONTEXT.md locked decision):**
7. In `applyInference` section 3, when an incoming risk fingerprint matches an existing risk that is `reviewed: true`, compare the incoming `signal` with the existing `signal`. If the incoming signal has different `affectedNodeIds` (different sorted join) or different `nodeId`, the risk has changed â€” set `reviewed: false` to resurface it. If the signal data is identical, keep the reviewed state.
8. After any resurface (setting reviewed to false), update `persistedReviewedIds` by removing that fingerprint and call `saveReviewedRisks` on the updated risks Map.

Pattern notes:
- Follow the existing double-paren Zustand pattern.
- Module-level `persistedReviewedIds` avoids reading localStorage on every applyInference call (only read on init).
- Keep localStorage writes synchronous (they are fast for small data).
  </action>

Introduce a new circular dependency for a previously-reviewed component â€” it should resurface as active. Check that `localStorage.getItem('archlens-reviewed-risks')` contains a valid JSON array of fingerprints.
  </verify>
  <done>Reviewed risk IDs survive page reload via localStorage. A risk that re-fires with different affected nodes resurfaces as active (reviewed=false). The `persistedReviewedIds` module-level set is initialized from localStorage on store creation.</done>
