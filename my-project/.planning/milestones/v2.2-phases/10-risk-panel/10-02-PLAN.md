  <name>Task 1: Verify and harden risk click-to-highlight integration</name>
  <files>packages/client/src/panels/RiskPanel.tsx, packages/client/src/App.tsx</files>
  <action>
Verify the end-to-end wiring from risk row click to canvas highlight + viewport pan, and harden edge cases.

**Multi-node risk handling:**
1. In `RiskItemRow`, the current click handler calls `onHighlightNode?.(risk.signal.nodeId)`. For circular dependency risks, `risk.signal.affectedNodeIds` may contain multiple node IDs. The current behavior (highlighting `nodeId`) is acceptable — `nodeId` is the primary offending node. No change needed IF `nodeId` is always set. Verify that `risk.signal.nodeId` is always a non-empty string for all risk types (circular_dependency, boundary_violation, excessive_fan_out). If it could be empty, fall back to `risk.signal.affectedNodeIds?.[0]`.
2. Add a safety check in the click handler: `const targetId = risk.signal.nodeId || risk.signal.affectedNodeIds?.[0]; if (targetId) onHighlightNode?.(targetId);` — this ensures that even if nodeId is empty, we still have a target for highlighting.

**Reviewed risk click prevention (verify from Plan 01):**
3. The existing `RiskItemRow` already gates the click with `if (!reviewed)`. After Plan 01's changes, verify this still works correctly — reviewed rows should not trigger highlight. No changes expected, just verification.

**App.tsx handleHighlightNode robustness:**
4. In `App.tsx`, `handleHighlightNode` calls `canvasRef.current?.selectNodeOnCanvas(nodeId)` and then reads `nodePositionsRef.current.get(nodeId)`. If the node has no position yet (e.g., it was just added and layout hasn't run), the pan won't happen. This is acceptable behavior (canvas stays where it is). Verify this doesn't cause any errors by checking the optional chaining is in place.
5. No changes needed in App.tsx if the optional chaining is already correct. The existing code uses `?.` on both `canvasRef.current` and checks `if (pos && viewportControllerRef.current)` before calling `panToNode`. This is already robust.

**Summary:** This task is primarily verification with a minor safety improvement to the click handler for multi-node risks. The heavy lifting was done in Phase 7.
