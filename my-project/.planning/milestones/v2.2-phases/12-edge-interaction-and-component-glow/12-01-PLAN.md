**EdgeRenderer.ts changes:**

1. Change `listening: false` to `listening: true` in `createLine()` on the Konva.Arrow constructor.

2. Add a wider invisible hit region for easier hover/click targeting. Set `hitStrokeWidth: 15` on each Arrow (Konva feature — extends the clickable area beyond the visible stroke without changing visual appearance).

3. Store `dependencyCount` as a custom attribute on each Arrow: `arrow.setAttr('dependencyCount', edge.dependencyCount ?? 1)` — needed for tooltip display.

4. Add a public method `getAllLines(): Map<string, Konva.Arrow>` that returns the internal `this.lines` map — needed by ArchCanvas for iterating edges during highlight/dim.

5. Add a public method `resetLineStyle(edgeId: string): void` that resets an arrow to its default stroke color and width (using `computeEdgeStrokeWidth` and `computeEdgeOpacity` from the edge's stored attributes). This is needed for clearing edge highlights.

**ArchCanvas.tsx changes:**

1. Add new props to `ArchCanvasProps`:
   - `onEdgeHover?: (data: { edgeId: string; sourceId: string; targetId: string; sourceName: string; targetName: string; dependencyCount: number; targetExports: string[]; x: number; y: number } | null) => void`

2. Keep a stable ref: `const onEdgeHoverRef = useRef(onEdgeHover); onEdgeHoverRef.current = onEdgeHover;`

3. Inside the main `useEffect`, add state tracking for highlighted edge:
   ```
   let highlightedEdgeId: string | null = null;
   ```

4. Add an `edge:mouseover` / `edge:mouseout` equivalent. Since Konva fires events on shapes, attach `mousemove` on the stage and check if the target is a Konva.Arrow. When a mousemove event fires on a Konva.Arrow:
   - Get sourceId and targetId from arrow attrs
   - Look up source and target node names from `graphStore.getState().nodes`
   - Get targetExports from the target node's `keyExports ?? []`
   - Get pointer position in screen coordinates: `stage.getPointerPosition()` gives canvas-relative coords
   - Call `onEdgeHoverRef.current?.({ edgeId, sourceId, targetId, sourceName, targetName, dependencyCount, targetExports, x: pointerPos.x, y: pointerPos.y })`
   - Track last hovered edge ID to avoid re-firing for the same edge.

5. When mousemove fires on a non-Arrow target (or on the stage background), call `onEdgeHoverRef.current?.(null)` to dismiss tooltip (only if previously showing).

6. Add edge click handler. Within the existing `stage.on('click tap')` handler, BEFORE the node-group walk-up logic, check if the clicked target is a Konva.Arrow (or has a Konva.Arrow parent). If so:
   - If clicking the same edge that's already highlighted, dismiss (call `clearEdgeHighlight()`)
   - Otherwise, call `highlightEdge(arrow)`:
     - Set `highlightedEdgeId = arrow.id()`
     - Change the clicked arrow's stroke to `'#60a5fa'` (accent blue) and strokeWidth to its current + 2
     - Find sourceId and targetId from arrow attrs
     - Call `highlightNode(sourceId, nodeRenderer)` and `highlightNode(targetId, nodeRenderer)` to highlight both endpoints
     - Dim all OTHER nodes: iterate `nodeRenderer.getAllNodeBounds()`, and for each node that is NOT sourceId and NOT targetId, set `shape.opacity(0.15)` (low opacity)
     - Smooth transition: Konva doesn't natively support CSS transitions, but the 200-300ms effect from CONTEXT.md can be achieved by directly setting opacity values (Konva redraws at 60fps so visual change is immediate; this is acceptable per existing codebase pattern)
   - `gl.batchDraw()`

7. Create `clearEdgeHighlight()` helper function:
   - If `highlightedEdgeId` is set, reset that arrow's style via `edgeRenderer.resetLineStyle(highlightedEdgeId)`
   - Call `clearSelection(nodeRenderer)` to restore all node opacities
   - Set `highlightedEdgeId = null`
   - `gl.batchDraw()`

8. Modify the existing background-click handler to also call `clearEdgeHighlight()` before `clearSelection()`.

9. Add Escape key dismissal for edge highlight: in the existing stage click handler area, add `stage.container().addEventListener('keydown', ...)` or better, within the effect, add a document-level keydown listener for Escape that calls `clearEdgeHighlight()` if an edge is highlighted. Clean up in the return function.

10. In the cleanup function, add: `stage.off('mousemove')` and the keydown listener removal.

**App.tsx changes:**

1. Add edge tooltip state:
   ```typescript
   const [edgeTooltip, setEdgeTooltip] = useState<{
     edgeId: string; sourceId: string; targetId: string;
     sourceName: string; targetName: string;
     dependencyCount: number; targetExports: string[];
     x: number; y: number;
   } | null>(null);
   ```

2. Pass `onEdgeHover={setEdgeTooltip}` to `<ArchCanvas>`.

3. Render an HTML tooltip overlay inside the canvas container div (position: absolute), conditionally rendered when `edgeTooltip` is not null:
   - Position: `left: edgeTooltip.x + 12`, `top: edgeTooltip.y - 8` (offset from cursor)
   - Style: dark card matching existing UI — `background: 'rgba(10, 10, 15, 0.95)'`, `border: '1px solid rgba(255, 255, 255, 0.15)'`, `borderRadius: 6`, `padding: '8px 12px'`, `color: '#ffffffe6'`, `fontSize: 12`, `fontFamily: 'monospace'`, `zIndex: 300`, `pointerEvents: 'none'`, `maxWidth: 280`
   - Content layout:
     - Line 1: `{sourceName}` in white, ` -> ` in gray, `{targetName}` in white
     - Line 2: `{dependencyCount} dependency/dependencies` (singular/plural) in dim gray
     - Line 3 (if targetExports.length > 0): Import symbols — show first 5 comma-separated, if >5 append `+ N more`. Label: "Imports: " in dim gray, symbols in slightly brighter color
   - Clamp position so tooltip doesn't overflow the canvas container boundaries (check against `dimensions.width` and `dimensions.height`)

**Important notes:**
- The GraphEdge type does NOT have an `importSymbols` field. Per CONTEXT.md "import symbols" decision, use the target node's `keyExports` as a proxy for what the source imports from the target. This is reasonable because component-level edges aggregate all file-level imports, and the target's exports are what the source depends on.
- Edge arrows currently use `moveToBottom()` pattern — ensure the new listening behavior doesn't interfere with node click-to-select (Konva hit detection checks shapes in z-order, so node groups on top should still receive clicks first).
- Per CONTEXT.md: "Smooth 200-300ms transition for the highlight/dim effect" — since the existing codebase sets opacity directly without CSS transitions (all Konva imperative), follow the same pattern. The visual change is immediate at 60fps which is acceptable.
