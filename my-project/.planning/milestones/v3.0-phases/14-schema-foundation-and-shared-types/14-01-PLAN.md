**1. Add table definitions to `packages/server/src/db/schema.ts`:**

Add `graphSnapshots` table after the existing `layoutPositions` table:
- `id`: integer, primary key, autoIncrement
- `sessionId`: text, notNull — session UUID for scoping
- `watchRoot`: text, notNull — watched directory path
- `sequenceNumber`: integer, notNull — monotonically increasing per session
- `timestamp`: integer with `mode: 'timestamp_ms'`, notNull
- `graphJson`: text with `mode: 'json'`, typed as `{ nodes: unknown[]; edges: unknown[]; positions: Record<string, { x: number; y: number }> }`, notNull — the full self-contained graph snapshot
- `summary`: text, notNull — human-readable summary like "15 nodes, 22 edges, 3 layers"
- `triggerFiles`: text with `mode: 'json'`, typed as `string[]`, notNull — file paths that triggered this snapshot
- `riskSnapshot`: text with `mode: 'json'`, typed as `unknown[]`, notNull, default `[]` — risk state at snapshot time

Add `intentSessions` table after `graphSnapshots`:
- `id`: integer, primary key, autoIncrement
- `sessionId`: text, notNull
- `watchRoot`: text, notNull
- `category`: text, notNull — IntentCategory enum value
- `objective`: text, notNull — human-readable intent label
- `confidence`: real, notNull — 0.0 to 1.0
- `subtasks`: text with `mode: 'json'`, typed as `string[]`, notNull, default `[]`
- `evidence`: text with `mode: 'json'`, typed as `string[]`, notNull, default `[]`
- `riskSnapshot`: text with `mode: 'json'`, typed as `unknown[]`, notNull, default `[]`
- `startSnapshotId`: integer, nullable — NO FK reference (foreign_keys=OFF per Pitfall 4 in RESEARCH.md)
- `endSnapshotId`: integer, nullable — NO FK reference
- `startedAt`: integer with `mode: 'timestamp_ms'`, notNull
- `endedAt`: integer with `mode: 'timestamp_ms'`, nullable

IMPORTANT: Do NOT use `.references()` on startSnapshotId/endSnapshotId — foreign_keys is OFF in connection.ts so FK constraints would be silently unenforced. Keep them as plain integer columns.

**2. Create `packages/shared/src/types/timeline.ts`:**

Follow the existing const-object + derived type pattern (see `inference.ts`). Use ASCII dash section separators.

Define `IntentCategory` const object with 6 values:
- `FEATURE_BUILDING: 'feature_building'`
- `BUG_FIXING: 'bug_fixing'`
- `REFACTORING: 'refactoring'`
- `TEST_WRITING: 'test_writing'`
- `INFRASTRUCTURE: 'infrastructure'`
- `UNCERTAIN: 'uncertain'`

Then derive the type: `export type IntentCategory = typeof IntentCategory[keyof typeof IntentCategory];`

Define domain interfaces:
- `SnapshotMeta`: id (number), sessionId (string), sequenceNumber (number), timestamp (number), summary (string), triggerFiles (string[])
- `IntentSession`: id (number), sessionId (string), category (IntentCategory), objective (string), confidence (number), subtasks (string[]), startSnapshotId (number | null), endSnapshotId (number | null), startedAt (number), endedAt (number | null)

Define 3 WebSocket message interfaces:
- `SnapshotSavedMessage`: `{ type: 'snapshot_saved'; meta: SnapshotMeta }`
- `IntentUpdatedMessage`: `{ type: 'intent_updated'; session: IntentSession }`
- `IntentClosedMessage`: `{ type: 'intent_closed'; sessionId: string; endSnapshotId: number | null }`

Add JSDoc comments on each interface following the existing style in inference.ts.

```
z.object({ id: z.number(), sessionId: z.string(), category: z.string(), objective: z.string(), confidence: z.number(), subtasks: z.array(z.string()), startSnapshotId: z.number().nullable(), endSnapshotId: z.number().nullable(), startedAt: z.number(), endedAt: z.number().nullable() })
```

- `SnapshotSavedMessageSchema`: `z.object({ type: z.literal('snapshot_saved'), meta: SnapshotMetaSchema })`

- `IntentClosedMessageSchema`: `z.object({ type: z.literal('intent_closed'), sessionId: z.string(), endSnapshotId: z.number().nullable() })`

Add all 3 to the `ServerMessageSchema` discriminated union array (now 8 members).

**1. Create `packages/server/src/db/repository/snapshots.ts`:**

Follow the exact pattern from `positions.ts`: import `eq`, `asc`, `desc`, `sql` from `drizzle-orm`, import `InferSelectModel`/`InferInsertModel` types, import `db` from `'../connection.js'`, import `graphSnapshots` from `'../schema.js'`.

Export types:
- `GraphSnapshotRow = InferSelectModel<typeof graphSnapshots>`
- `GraphSnapshotInsert = InferInsertModel<typeof graphSnapshots>`

Export `snapshotsRepository` plain object with these methods:

- `insert(snapshot: Omit<GraphSnapshotInsert, 'id'>): number` — insert row, return `Number(result.lastInsertRowid)`
- `findById(id: number): GraphSnapshotRow | undefined` — select by id, use `.get()`
- `findBySession(sessionId: string): GraphSnapshotRow[]` — select all for session, order by `asc(sequenceNumber)`, use `.all()`
- `getMetaBySession(sessionId: string): Array<{ id: number; sessionId: string; sequenceNumber: number; timestamp: Date; summary: string; triggerFiles: string[] }>` — select only metadata columns (no graphJson), order by `asc(sequenceNumber)`. Use Drizzle `.select({ id: graphSnapshots.id, sessionId: graphSnapshots.sessionId, sequenceNumber: graphSnapshots.sequenceNumber, timestamp: graphSnapshots.timestamp, summary: graphSnapshots.summary, triggerFiles: graphSnapshots.triggerFiles })` to avoid loading the large JSON blob when listing snapshots for timeline browsing.
- `getCount(sessionId: string): number` — count rows for session using `sql<number>\`count(*)\``
- `deleteOldest(sessionId: string): void` — find single oldest by `asc(sequenceNumber)` with `limit(1)`, delete by id. Guard with `if (oldest)`.
- `deleteBySession(sessionId: string): void` — delete all rows matching sessionId
- `deleteByWatchRoot(watchRoot: string): void` — delete all rows matching watchRoot

**2. Create `packages/server/src/db/repository/intentSessions.ts`:**

Same import pattern. Export types:
- `IntentSessionRow = InferSelectModel<typeof intentSessions>`
- `IntentSessionInsert = InferInsertModel<typeof intentSessions>`

Export `intentSessionsRepository` plain object with these methods:

- `insert(session: Omit<IntentSessionInsert, 'id'>): number` — insert, return lastInsertRowid as number
- `findById(id: number): IntentSessionRow | undefined` — select by id
- `findBySession(sessionId: string): IntentSessionRow[]` — all sessions for a session, order by `asc(startedAt)`
- `findActive(sessionId: string): IntentSessionRow | undefined` — find row where `endedAt` is null and sessionId matches. Use `db.select().from(intentSessions).where(and(eq(intentSessions.sessionId, sessionId), isNull(intentSessions.endedAt))).get()`. Import `and`, `isNull` from `drizzle-orm`.
- `close(id: number, endSnapshotId: number | null, endedAt: Date): void` — update row by id, set endSnapshotId and endedAt
- `deleteBySession(sessionId: string): void` — delete all rows matching sessionId
- `deleteByWatchRoot(watchRoot: string): void` — delete all rows matching watchRoot
