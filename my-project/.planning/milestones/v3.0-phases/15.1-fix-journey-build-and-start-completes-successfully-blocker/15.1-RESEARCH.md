### Anti-Patterns to Avoid

- **Writing files manually instead of restoring from git:** The HEAD versions are correct and tested. Rewriting from scratch risks introducing diffs. Always use `git checkout HEAD --` for this class of regression.
- **Restoring from wrong git path:** The repo root is `..` relative to `my-project/`. Files in git are at `my-project/.auto-gsd/...`, not `.auto-gsd/...`. Using the wrong path prefix causes "pathspec did not match any file" errors.
- **Running `npm install` unnecessarily:** `@playwright/test` 1.58.2 is already installed in `.auto-gsd/journey-tests/node_modules/`. Reinstalling wastes time and may change the lock file unintentionally.
- **Reinstalling Chromium:** Browser is already installed in the ms-playwright LOCALAPPDATA cache. No reinstall needed.
- **Running build before tests:** `pnpm build` is tested within the journey suite itself. Build dist output already exists. No pre-run build needed to run the tests.

## Common Pitfalls

### Pitfall 1: Auto-gsd Overwrites Files Again After Fix

**What goes wrong:** The fix is applied, tests pass, then auto-gsd runs the next phase and overwrites the files again, causing the same regression.

**Why it happens:** Auto-gsd generates journey test files as part of its orchestration. It has templates for these files that produce simplified stubs.

**How to avoid:** Per the decisions made in Phase 14.3: "Keep files committed to git. If overwritten by auto-gsd, the fix is to restore from git." The committed state is the source of truth. Accept that auto-gsd may overwrite files; the `git checkout HEAD --` restore command is the repeatable fix.

**Warning signs:** `git status` shows `M` next to spec files or `D` next to `journey-phase-15.spec.ts` after auto-gsd runs. The `results.json` shows all tests as "skipped" with 0ms duration.

### Pitfall 2: Git Path Prefix Error

**What goes wrong:** `git checkout HEAD -- .auto-gsd/journey-tests/playwright.config.ts` fails with "pathspec did not match any file known to git".

**Why it happens:** The git repository root is at `F:\Directory\FrontierIntelligenceHack\` (one level above `my-project/`). All files are tracked with the `my-project/` prefix in git. Running from inside `my-project/` with relative paths like `.auto-gsd/...` fails because git does not find those paths in the tree.

**How to avoid:** Use the correct path prefix. From inside `my-project/`, use `git -C .. checkout HEAD -- my-project/.auto-gsd/...`. Or `cd ..` first and then run with `my-project/` prefix paths.

**Warning signs:** `fatal: pathspec '.auto-gsd/journey-tests/playwright.config.ts' did not match any file(s) known to git`

### Pitfall 3: Results Show "Skipped" Rather Than "Failed"

**What goes wrong:** The `results.json` shows `"status": "skipped"` for all 12 tests with 0ms duration. This looks like Playwright ran but tests were explicitly skipped, when in fact the test runner infrastructure itself is broken.

**Why it happens:** When Playwright can't bootstrap the test runner (e.g., the config file lacks a `webServer` block and no dev server is running), it reports tests as "skipped" rather than "errored". The stub `playwright.config.ts` lacks the webServer block, so tests that require a live server are effectively skipped.

**How to avoid:** Check for the webServer block in `playwright.config.ts` first. If missing, the restore from HEAD will add it back.

**Warning signs:** `results.json` has `"skipped": 12, "expected": 0, "unexpected": 0` with near-zero `duration`.

### Pitfall 4: Node Modules Version Mismatch After Package Manager Upgrade

**What goes wrong:** After restoring files, `npx playwright test` fails because the installed `@playwright/test` version (1.58.2) doesn't match what the spec files expect.

**Why it happens:** The `package.json` specifies `^1.50.0` which allows any `1.x` version. The installed version is 1.58.2. If the spec files use APIs only available in 1.58.x, an older installed version would fail.

**How to avoid:** The spec files use standard Playwright APIs (`page.request`, `test.setTimeout`, `expect`, etc.) that are stable across 1.50.x–1.58.x. No version mismatch expected. If issues occur, verify with `node node_modules/@playwright/test/package.json` from `.auto-gsd/journey-tests/`.

**Warning signs:** `TypeError: page.request.post is not a function` or similar API-not-found errors.
