<decisions>
## Implementation Decisions

### Test data strategy
- Trigger real file changes to generate snapshot/intent data (no diagnostic seeding endpoints)
- Use dedicated temp directories per test — create via POST /api/watch to switch the server's watch root
- Each test gets a fresh temp directory for full isolation (pipeline restarts on watch root switch)
- Poll /api/timeline until snapshots appear (with timeout) rather than fixed delays — faster on fast machines, still bounded

### Assertion depth
- **GET /api/timeline**: Verify shape (id, sequenceNumber, timestamp, summary, triggerFiles) AND monotonically increasing sequenceNumbers
- **GET /api/snapshot/:id**: Deep validation — node IDs match written files, edges reflect real imports, positions have numeric x/y for EVERY node (Object.keys(positions).length === nodes.length)
- **GET /api/intents**: Full shape validation — category is valid IntentCategory enum, confidence between 0 and 1, subtasks is array, startSnapshotId is number, objective is string, sessionId matches
- **Timestamps**: Validate epoch millisecond range (> 1735689600000 for 2025-01-01, < reasonable future bound)
- **Error cases**: Include 404 for missing snapshot IDs and 400 for invalid (non-numeric) IDs

### Concurrency verification
- Parallel write + read approach: while writing 10+ files to trigger snapshots, simultaneously call GET /api/timeline and verify it responds within 5 seconds
- After the parallel phase, poll /api/timeline to confirm new snapshots appear from the written files — proves pipeline didn't drop events
- 5-second timeout for concurrent API reads (lenient for CI environments)

### Test runner approach
- Use Playwright's APIRequestContext (request fixture) for HTTP calls — no extra dependencies
- Use Playwright baseURL from config (not manual BASE_URL env var)
- Mixed fixture usage: `request` for API calls + `page` for basic browser load checks
- Create temp directories using Node.js fs/os imports directly (fs.mkdtempSync, os.tmpdir) — simple, no abstraction layer

### Claude's Discretion
- Exact file contents written to trigger snapshots (as long as they create valid imports/dependencies)
- Poll interval and max retries for waiting on snapshots
- Test naming and describe block structure
- Cleanup strategy for temp directories (afterEach vs afterAll)

</decisions>
