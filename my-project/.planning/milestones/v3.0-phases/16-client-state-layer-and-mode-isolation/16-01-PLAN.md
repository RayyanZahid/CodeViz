Output: A new `replayStore.ts` Zustand slice with mode state and buffer, modified `wsClient.ts` that gates message dispatch through replay check, and DirectoryBar auto-exit-replay on watch-root switch.

<task type="auto">
  <name>Task 1: Create replayStore Zustand slice with mode state and buffer</name>
  <files>packages/client/src/store/replayStore.ts</files>
  <action>
Create `packages/client/src/store/replayStore.ts` following the exact same double-paren Zustand pattern as `graphStore.ts` and `inferenceStore.ts`.

**State fields:**
- `isReplay: boolean` (default false) — the master mode gate
- `replaySnapshotId: number | null` (default null) — current snapshot ID being viewed
- `replayTimestamp: number | null` (default null) — epoch ms of the replayed point, displayed in banner
- `replayNodes: Map<string, GraphNode>` — the historical graph nodes for replay rendering
- `replayEdges: Map<string, GraphEdge>` — the historical graph edges for replay rendering
- `bufferedGraphDeltas: GraphDeltaMessage[]` — live graph_delta messages buffered during replay
- `bufferedInferenceMessages: InferenceMessage[]` — live inference messages buffered during replay
- `bufferedEventCount: number` — total count of buffered events (for banner display efficiency — avoids array.length selector re-renders)
- `bufferOverflowed: boolean` (default false) — set true when buffer exceeds 500 entries

**Actions:**
- `enterReplay(snapshotId: number, timestamp: number, nodes: GraphNode[], edges: GraphEdge[]): void`
  - Sets isReplay=true, stores snapshot data into Maps, clears buffers, resets bufferOverflowed
- `exitReplay(): void`
  - Sets isReplay=false, clears replayNodes/replayEdges/snapshotId/timestamp
  - Does NOT clear buffers — caller reads bufferedGraphDeltas/bufferedInferenceMessages before calling exitReplay, then calls clearBuffer after draining
- `bufferGraphDelta(msg: GraphDeltaMessage): void`
  - If buffer length >= 500, set bufferOverflowed=true and skip push
  - Otherwise push to bufferedGraphDeltas, increment bufferedEventCount
- `bufferInference(msg: InferenceMessage): void`
  - Push to bufferedInferenceMessages, increment bufferedEventCount
- `clearBuffer(): void`
  - Clears both buffer arrays, resets bufferedEventCount to 0, resets bufferOverflowed to false

**Imports:**
- `import { create } from 'zustand';`
- `import type { GraphNode, GraphEdge } from '@archlens/shared/types';`
- `import type { GraphDeltaMessage, InferenceMessage } from '@archlens/shared/types';`

**Exports:**
- `export const useReplayStore = create<ReplayStore>()(...)` — React hook
- `export const replayStore = useReplayStore;` — vanilla reference (same pattern as graphStore)
- `export interface ReplayStore { ... }` — for type imports

The 500-message buffer cap per RESEARCH.md Open Question 3 prevents memory issues during long replay sessions. When `bufferOverflowed` is true, exit-replay uses snapshot fetch instead of buffer drain.
