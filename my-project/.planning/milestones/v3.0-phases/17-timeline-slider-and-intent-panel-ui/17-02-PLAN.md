  <action>
Create `packages/client/src/panels/IntentPanel.tsx` following the exact collapsible panel pattern from RiskPanel.tsx (same styling conventions, inline styles only, no CSS framework).

**Panel structure:**

1. **Header row** (always visible, clickable to collapse/expand):
   - Triangle indicator (collapsed: ▶, expanded: ▼)
   - "INTENT" label (uppercase monospace, same style as Risk panel header)
   - Confidence badge — colored pill next to label:
     - green (#22c55e) when confidence >= 0.7
     - amber (#eab308) when confidence >= 0.4
     - red (#ef4444) when confidence < 0.4
     - Badge text: percentage (e.g., "85%"), format: `Math.round(confidence * 100) + '%'`
   - Only show confidence badge when activeSession is not null

2. **Objective label** (first section inside panel body):
   - Display `activeSession.objective` string (server provides human-readable text like "Adding Feature", "Bug Fix")
   - Below objective: category tag in muted style (e.g., "feature_building" formatted as "Feature Building" via simple string transform: split on '_', capitalize each word, join with space)
   - When no active session: show empty state: "No activity detected" in muted text

3. **Subtask checklist** (second section):
   - Derive subtasks client-side from `inferenceStore.activityFeed` items:
     - Group activityFeed items by their `iconColor` (which maps to event categories):
       - #22c55e (green) = "File creation"
       - #f97316 (orange) = "Risk detection"
       - #3b82f6 (blue) = "Dependency changes"
       - #94a3b8 (gray) = "File modifications"
     - Each group with > 0 items becomes a subtask entry
     - Show count next to each: "File creation (3)" with a checkmark if count > 0
   - If activityFeed is empty, hide the subtasks section entirely
   - Use `useMemo` to derive subtasks from activityFeed for performance

4. **Focus-shift notification** (third section, conditional):
   - Check intentHistory: if the most recent closed session has a different category than the activeSession, show a focus-shift notice:
     - Text: "Switched from {prevCategory} to {currentCategory}" where categories are formatted (split on '_', capitalize)
     - Styled as a small amber (#eab308) notice row with left border accent
   - Only show when both activeSession and intentHistory[0] exist and differ in category

5. **Risk correlation** (fourth section, conditional):
   - Read `risks` from `useInferenceStore(s => s.risks)`
   - Count unreviewed risks: `[...risks.values()].filter(r => !r.reviewed).length`
   - If unreviewed > 0, show a red badge: "⚠ {count} active risk(s)" with red (#ef4444) text
   - If unreviewed === 0, hide this section

6. **Intent history log** (fifth section, collapsible):
   - Sub-header: "History" with its own collapse toggle
   - List intentHistory entries (newest first, max 10 shown):
     - Each row: category (formatted), objective text, timestamp (format using Intl.DateTimeFormat for "2:15 PM" style, same pattern as ReplayBanner.tsx)
     - Muted styling, compact rows
   - If intentHistory is empty, show "No history yet" in muted text

**State management:**
- `const [collapsed, setCollapsed] = useState(false)` — main panel collapse
- `const [historyCollapsed, setHistoryCollapsed] = useState(true)` — history sub-section collapse (starts collapsed)
- Read activeSession and intentHistory from `useIntentStore`
- Read activityFeed from `useInferenceStore(s => s.activityFeed)` for subtask derivation
- Read risks from `useInferenceStore(s => s.risks)` for risk correlation

**Styling:** Follow RiskPanel.tsx exactly — transparent background, borderTop separator, monospace fonts, inline styles, same padding/gap/fontSize values.

**Import from intentStore.ts** — use: `import { useIntentStore } from '../store/intentStore.js';`
Note: intentStore.ts does not exist yet (Plan 01 creates it). If developing independently, create a minimal stub with the interface. The executor will handle dependency ordering.
