    **1. Create `packages/server/src/parser/extractors/python.ts`:**

    Export function `extractPython(rootNode: Parser.SyntaxNode, filePath: string): { imports: ImportInfo[], exports: ExportInfo[], calls: CallInfo[] }`

    **Python import extraction:**
    - Walk `rootNode.namedChildren` for node types: `import_statement` and `import_from_statement`
    - `import_statement`: `import foo` — extract module name from `dotted_name` child
    - `import_from_statement`: `from foo import bar` — extract the `module_name` child (the `from` source)
    - Handle relative imports: `from . import X` and `from ..utils import Y` — preserve the dots in the source string
    - Python has no type-only imports — set `isTypeOnly: false` always

    **Python export extraction (top-level definitions):**
    - Python has no explicit export keyword. For ArchLens, treat top-level definitions as "exports" since they are importable by other modules.
    - Walk `rootNode.namedChildren` for: `function_definition`, `class_definition`, `assignment` at module level
    - `function_definition`: extract `name` child text
    - `class_definition`: extract `name` child text
    - `assignment`: extract the left-hand `identifier` text (e.g., `MY_CONSTANT = ...`)
    - Set `isDefault: false` always (Python has no default export concept)
    - Set `isTypeOnly: false` always

    **Python call extraction:**
    - Walk all nodes for `call` node type (Python tree-sitter uses `call`, not `call_expression`)
    - Extract callee: the `function` child of the `call` node
    - If callee is `identifier`, use its text (direct call like `print()`)
    - Skip `attribute` callees (method calls like `self.method()` — too noisy for v1)
    - `isTopLevel`: true if not inside a `function_definition` or `class_definition` body

    **2. Update `packages/server/src/parser/worker.ts`:**

    - Add import: `import { extractPython } from './extractors/python.js';`
    - Add Python parser to grammar cache: `import Python from 'tree-sitter-python';`
    - In the language dispatch switch, add case for `'py'`: use Python parser + extractPython extractor
    - Ensure `tsconfig.workers.json` `include` covers the new python.ts file (it should already since it includes `src/parser/**/*.ts`)

    **3. Rebuild workers:** Run `pnpm --filter @archlens/server build:workers` to verify the updated worker compiles with the Python extractor included.

  <action>
    **Create `packages/server/src/pipeline/Pipeline.ts`:**

    **Class: Pipeline**
    The Pipeline is the integration orchestrator. It receives batches from FileWatcher, reads file contents, creates ParseTasks, dispatches to ParserPool, and emits ParseBatchResults.

    **Constructor takes:**
    - `watchRoot: string` — absolute path to watched directory
    - `onResult: (result: ParseBatchResult) => void` — callback for downstream consumers (Phase 3 will plug into this)
    - Optional `options: { debounceMs?: number, minThreads?: number, maxThreads?: number }`

    **Internal components (created in constructor):**
    - `this.watcher = new FileWatcher(watchRoot, (batch) => this.handleBatch(batch), { debounceMs: options?.debounceMs })`
    - `this.parserPool = new ParserPool({ minThreads: options?.minThreads, maxThreads: options?.maxThreads })`
    - `this.sequenceCounter = 0` — global monotonic counter

    **`handleBatch(batch: FileWatchBatch): void` method:**

    1. Separate events into two groups: removals (`type === 'removed'`) and parse-needed (`type === 'added' | 'modified'`)

    2. For removals:
       - Create `FileRemoved` objects with `{ filePath, sequenceId: this.sequenceCounter++, type: 'removed' }`
       - Call `this.parserPool.evictFile(filePath)` for each removed file (clean tree cache)

    3. For parse-needed events:
       - Read file contents: `await fs.readFile(path.join(this.watchRoot, event.relativePath), 'utf-8')`
       - Determine language from file extension using a utility: `.ts` -> `'ts'`, `.tsx` -> `'tsx'`, `.js` -> `'js'`, `.jsx` -> `'jsx'`, `.py` -> `'py'`
       - Create `ParseTask` objects: `{ filePath: event.relativePath, source, language, sequenceId: this.sequenceCounter++ }`
       - Handle read errors gracefully: if file was deleted between event and read, treat as removal; if encoding error or binary, skip with warning

    4. Dispatch parse tasks: `const results = await this.parserPool.parseBatch(tasks)`

    5. Combine results: merge parse results and removal results into a single `ParseBatchResult`:
       ```typescript
       { results: [...removedResults, ...parseResults], batchSequenceStart: batch.sequenceStart, processedAt: Date.now() }
       ```

    6. Call `this.onResult(batchResult)`

    **Lifecycle methods:**
    - `async start(): Promise<void>` — calls `this.watcher.start()`. Log "Pipeline started, watching {watchRoot}"
    - `async stop(): Promise<void>` — calls `this.watcher.stop()` then `this.parserPool.destroy()`. Log "Pipeline stopped"
    - `isRunning(): boolean` — returns `this.watcher.isWatching()`

    **Error handling:**
    - Wrap handleBatch in try/catch — log errors but don't crash the pipeline
    - Individual file read failures don't abort the entire batch — skip the file and continue

    **Language detection helper (private method or module-level function):**
    ```typescript
    const EXTENSION_TO_LANGUAGE: Record<string, SupportedLanguage> = {
      '.ts': 'ts', '.tsx': 'tsx', '.js': 'js', '.jsx': 'jsx', '.py': 'py',
    };
    function detectLanguage(filePath: string): SupportedLanguage | null {
      const ext = path.extname(filePath);
      return EXTENSION_TO_LANGUAGE[ext] ?? null;
    }
    ```

    Import: `FileWatcher` from `../watcher/FileWatcher.js`, `ParserPool` from `../parser/ParserPool.js`, `fs` from `node:fs/promises`, `path` from `node:path`, types from `@archlens/shared`.
