  <action>
Create `packages/server/src/graph/DependencyGraph.ts` — the core graph module.

**Class structure:** `DependencyGraph extends EventEmitter<{ delta: [delta: GraphDelta] }>`

**Instance fields:**
- `private readonly g: Graph` — `new Graph({ directed: true })` from `@dagrejs/graphlib`. Note: graphlib v4 Graph constructor takes `{ directed }`, node/edge labels are set via setNode/setEdge. Use type assertions for label types since graphlib's generics may not fully support them.
- `private version: number = 0` — monotonic delta version counter
- `private prevFileResults: Map<string, ParseResult> = new Map()` — previous parse results per file for incremental diffing
- `private activeCycles: Set<string> = new Set()` — canonicalized cycle strings for diff detection
- `private pendingBatches: ParseBatchResult[] = []` — accumulator for batch consolidation
- `private consolidateTimer: ReturnType<typeof setTimeout> | null = null`
- `private readonly CONSOLIDATE_MS = 50` — consolidation window (Claude's discretion per CONTEXT.md)

**Public methods:**

1. `onParseResult(batch: ParseBatchResult): void` — Entry point from Pipeline. Pushes batch to pendingBatches, clears/resets consolidateTimer with setTimeout to call flushPending() after CONSOLIDATE_MS.

2. `getVersion(): number` — Returns current version.

3. `getNodeCount(): number` — Returns `g.nodeCount()`.

4. `getEdgeCount(): number` — Returns `g.edgeCount()`.

5. `hasNode(id: string): boolean` — Returns `g.hasNode(id)`.

6. `getNodeMetadata(id: string): NodeMetadata | undefined` — Returns `g.node(id)`.

**Private methods:**

1. `flushPending(): void` — Drains pendingBatches, collects trigger files, calls applyBatches.

2. `applyBatches(batches: ParseBatchResult[], triggerFiles: string[]): void` — Main update logic:
   - Create a mutable delta builder object
   - For each batch, for each result: if `'type' in result && result.type === 'removed'`, call processRemoval; else call processFile
   - After all files processed, detect cycle changes
   - Increment version, build final GraphDelta
   - Emit 'delta' event with the GraphDelta
   - Store the delta for persistence (call a protected `onDeltaComputed(delta)` hook that Plan 02 will override/extend)

3. `processFile(result: ParseResult, delta: MutableDelta): void` — Per-file incremental update:
   - Node ID = `result.filePath` (project-relative path — already normalized by Pipeline)
   - Upsert node: if new, `g.setNode(nodeId, buildNodeMetadata(result))`, add to `delta.addedNodes`; if exists and exports changed (compare sorted export names vs prev), update label, add to `delta.modifiedNodes`; if exists and unchanged, skip
   - Compute new edges from `result.imports`: for each ImportInfo, resolve the target:
     - If `source` starts with `.` or `..`: resolve relative to `path.dirname(result.filePath)` using `path.posix.normalize(path.posix.join(...))`, add `.ts` extension if missing and no extension present. Use posix paths throughout since FileWatcher normalizes to forward slashes.
     - If `source` starts with `node:` or is a bare specifier (no `.` prefix): create external node ID `__ext__/${source}`
   - Build `Map<targetId, string[]>` of new edges (aggregate symbols per target — per locked decision, single edge per file pair)
   - Compare with `g.outEdges(nodeId)`: remove edges no longer present, add new edges. Track in delta.
   - Update prevFileResults

4. `processRemoval(filePath: string, delta: MutableDelta): void`:
   - If node exists, record all incident edges in delta.removedEdgeIds
   - `g.removeNode(filePath)` — graphlib automatically removes incident edges
   - Add to delta.removedNodeIds
   - Delete from prevFileResults

5. `detectCycleChanges(delta: MutableDelta): void`:
   - Run `alg.findCycles(this.g)` — returns `string[][]`
   - Canonicalize each cycle: rotate so lexicographically smallest node ID is first, join with ` -> `
   - Diff against activeCycles to find added/removed cycles
   - For added cycles, compute severity via in-degree sum: `high` >= 10, `medium` >= 4, `low` < 4 (thresholds from RESEARCH.md recommendation)
   - For removed cycles, use 'low' severity (the cycle is gone, severity is informational)
   - Update activeCycles set

6. `buildNodeMetadata(result: ParseResult): NodeMetadata` — Helper to create NodeMetadata from ParseResult.

**Import resolution helper:** `resolveImportTarget(fromFile: string, importSource: string): string` — encapsulates the relative vs external resolution logic described above.

**Important implementation notes:**
- Use `import { Graph, alg } from '@dagrejs/graphlib';` (named ESM imports, verified in research)
- Use `import { EventEmitter } from 'node:events';` with typed event map
- Import types from `@archlens/shared/types` (project convention)
- Use .js extensions on all relative imports
- Do NOT use `alg.topsort` for cycle detection (throws on cycles — use `alg.findCycles`)
- Run cycle detection ONCE after all files in the batch, not per-file
- The `onDeltaComputed` hook is a protected method that defaults to no-op — Plan 02's persistence layer will extend this
