  <name>Task 1: Implement EventCorroborator with signal counter and threshold-based event firing</name>
  <files>packages/server/src/inference/EventCorroborator.ts</files>
  <action>
Create `packages/server/src/inference/EventCorroborator.ts` implementing the multi-signal corroboration mechanism.

**Core data structure:**
- `counters: Map<string, { count: number; deltaVersion: number }>` — keyed by canonical "candidate event" string.
- `THRESHOLD = 2` — minimum signals before an event fires (locked decision).
- `MAX_STALE_VERSIONS = 10` — evict counter entries older than 10 delta versions (prevents unbounded memory growth, per RESEARCH.md Pitfall 2).

**processDelta(delta: GraphDelta, zoneUpdates: ZoneUpdate[]): ArchitecturalEvent[]:**

The method processes each delta to accumulate corroboration signals and fire events when threshold is met.

1. **Eviction pass:** Before processing, iterate counters and delete any entry where `currentVersion - entry.deltaVersion > MAX_STALE_VERSIONS`. Also delete counters keyed on any nodeId in `delta.removedNodeIds`.

2. **component_created signals:** For each nodeId in `delta.addedNodes`, create key `component_created:${nodeId}`. Increment counter. If count >= THRESHOLD, fire event with type 'component_created', nodeId, zone from zoneUpdates, timestamp from delta.timestamp. Delete the counter entry on fire.

3. **dependency_added signals:** For each edge in `delta.addedEdges`, create key `dependency_added:${edge.v}:${edge.w}`. Same counter/fire logic.

4. **dependency_removed signals:** For each edgeId in `delta.removedEdgeIds`, create key `dependency_removed:${edgeId}`. Same counter/fire logic. Parse nodeId and targetNodeId from the edgeId string (format: `sourceId->targetId`).

5. **Split/merge detection:**
   - **component_merged:** If `delta.removedNodeIds.length >= 2` AND `delta.addedNodes.length === 1`, create key `component_merged:${delta.addedNodes[0]}`. Counter/fire logic.
   - **component_split:** If `delta.removedNodeIds.length === 1` AND `delta.addedNodes.length >= 2`, create key `component_split:${delta.removedNodeIds[0]}`. Counter/fire logic.

**Counter increment helper:**
```typescript
private incrementCounter(key: string, deltaVersion: number): boolean {
  const entry = this.counters.get(key);
  const newCount = (entry?.count ?? 0) + 1;
  if (newCount >= this.THRESHOLD) {
    this.counters.delete(key);
    return true; // fires
  }
  this.counters.set(key, { count: newCount, deltaVersion });
  return false; // accumulating
}
```

**Critical locked decisions:**
- Binary pass/fail — events either fire or don't, no confidence scores exposed.
- Events fire immediately at threshold — no time-window batching.
- A single file edit produces one delta, so the counter reaches 2 only after TWO deltas with the same signal. This fulfills "a single file edit must never trigger an architectural event."
- Counter is deleted (not zeroed) when an event fires — re-accumulation starts fresh.

<task type="auto">
  <name>Task 2: Implement RiskDetector with cycle enrichment, boundary violations, and fan-out detection</name>
  <files>packages/server/src/inference/RiskDetector.ts</files>
  <action>
Create `packages/server/src/inference/RiskDetector.ts` implementing three independent risk detection algorithms.

**Exported constants:**
- `ZONE_LAYER_ORDER: Partial<Record<ZoneName, number>>` — `{ frontend: 0, api: 1, services: 2, 'data-stores': 3 }`. Infrastructure and external are NOT in the ordering (not subject to layering rules).
- `FAN_OUT_THRESHOLD = 8` — locked decision: more than 8 outgoing internal dependencies = warning.

**RiskDetector class:**

**detectRisks(delta: GraphDelta, getZone: (nodeId: string) => ZoneName, getOutEdges: (nodeId: string) => GraphDeltaEdge[]): RiskSignal[]:**

1. **Circular dependency risks** (wraps Phase 3 cycle detection — no independent re-detection):
   - For each cycle in `delta.cyclesAdded`:
     - Get zone label for each node in the cycle path using `getZone(nodeId)`.
     - Create RiskSignal with type 'circular_dependency', severity 'critical' (locked), nodeId = cycle.path[0], affectedNodeIds = cycle.path, details = descriptive string including zone labels.

2. **Boundary violation risks** (check newly added edges for layer-skip):
   - For each edge in `delta.addedEdges`:
     - Skip if either endpoint starts with `__ext__/` (external nodes are not subject to layering per RESEARCH.md anti-pattern).
     - Look up `sourceLayer = ZONE_LAYER_ORDER[getZone(edge.v)]` and `targetLayer = ZONE_LAYER_ORDER[getZone(edge.w)]`.
     - If either is undefined (node is in infrastructure, external, or unknown zone), skip — no boundary rule applies.
     - Compute `diff = targetLayer - sourceLayer`.
     - If `diff !== 1 && diff !== 0`, it's a violation:
       - `diff > 1`: forward layer skip (e.g., frontend -> data-stores)
       - `diff < 0`: backward import (e.g., services -> frontend)
       - `diff === 0`: same zone — allowed
       - `diff === 1`: adjacent layer — allowed
     - Create RiskSignal with type 'boundary_violation', severity 'warning' (locked), details describing the violation direction.

3. **Fan-out risks** (check nodes whose out-degree may have changed):
   - Collect unique nodeIds from: `delta.addedNodes`, `delta.modifiedNodes`, and source nodes (`edge.v`) from `delta.addedEdges`.
   - For each nodeId, call `getOutEdges(nodeId)` and count only edges where `!edge.w.startsWith('__ext__/')` (internal edges only — per RESEARCH.md Pitfall 6, external stubs inflate out-degree and cause false positives on server entry points).
   - If internal out-degree > FAN_OUT_THRESHOLD (8), create RiskSignal with type 'excessive_fan_out', severity 'warning' (locked), details including actual count vs threshold.

**Important:** RiskDetector does NOT maintain state across deltas — it's a pure function of each delta + current zone assignments. This keeps it simple and testable.
