**2. Add `getSnapshot()` to `packages/server/src/graph/DependencyGraph.ts`:**

Add a public method that serializes the current in-memory graph into the `GraphNode[]` and `GraphEdge[]` arrays matching the shared `GraphNode` and `GraphEdge` interfaces from `graph.ts`. This method is read-only â€” no mutations.

```typescript
import type { GraphNode, GraphEdge } from '@archlens/shared/types';

getSnapshot(): { nodes: GraphNode[]; edges: GraphEdge[] } {
  const nodes: GraphNode[] = [];
  for (const id of this.g.nodes()) {
    // Skip external stub nodes â€” they have no metadata and are not meaningful for the client
    if (id.startsWith('__ext__/')) continue;
    const meta = this.g.node(id) as NodeMetadata | undefined;
    if (!meta) continue;
    nodes.push({
      id,
      name: id.split('/').pop()?.replace(/\.\w+$/, '') ?? id,
      nodeType: 'service_module', // Default â€” Phase 4 zone classification enriches this
      zone: null, // Zone will be filled by InferenceEngine zone updates
      fileList: [meta.filePath],
      incomingEdgeCount: this.g.inEdges(id)?.length ?? 0,
      outgoingEdgeCount: this.g.outEdges(id)?.length ?? 0,
      lastModified: new Date(meta.lastModified),
    });
  }

  const edges: GraphEdge[] = [];
  for (const e of this.g.edges()) {
    // Skip edges involving external stubs
    if (e.v.startsWith('__ext__/') || e.w.startsWith('__ext__/')) continue;
    edges.push({
      id: `${e.v}->${e.w}`,
      sourceId: e.v,
      targetId: e.w,
      edgeType: 'imports_depends_on',
    });
  }

  return { nodes, edges };
}
```

Import `GraphNode` and `GraphEdge` types at the top of the file alongside existing imports. The `NodeType` value `'service_module'` is the safe default â€” the client uses zone information from InferenceMessage for visual grouping, not nodeType.

**1. Create `packages/server/src/plugins/websocket.ts`:**

Build a Fastify plugin that:
- Registers `@fastify/websocket` as a sub-plugin
- Maintains a `Set<WebSocket>` of connected clients at module level
- Subscribes to `graph.on('delta', ...)` ONCE at plugin registration time (NOT inside the per-connection route handler â€” this avoids O(N^2) listener leak per RESEARCH.md Pitfall 4)
- Subscribes to `inferenceEngine.on('inference', ...)` ONCE at plugin registration time
- Broadcasts delta events as `GraphDeltaMessage` by mapping from internal `GraphDelta` (which uses `addedNodes: string[]` node IDs) to the wire format `GraphDeltaMessage` (which uses `addedNodes: GraphNode[]` objects). Use `graph.getNodeMetadata()` and `graph.getSnapshot()` logic to build `GraphNode` objects for the delta.
- Broadcasts inference events as `InferenceMessage`
- Registers a `/ws` route with `{ websocket: true }`. The route handler receives the WebSocket directly (v10+ API â€” NOT `connection.socket` pattern).
  - On connect: add socket to client set, send full `InitialStateMessage` via `graph.getSnapshot()` + `graph.getVersion()`
  - On close: remove socket from client set
  - On message from client: no-op for now (Phase 5 is server-to-client only)

The `broadcast` function must check `socket.readyState === socket.OPEN` before sending (per RESEARCH.md anti-pattern).

Plugin signature following existing `healthPlugin` pattern:
```typescript
import type { FastifyPluginAsync } from 'fastify';
import type { DependencyGraph } from '../graph/DependencyGraph.js';
import type { InferenceEngine } from '../inference/InferenceEngine.js';

export const websocketPlugin: FastifyPluginAsync<{
  graph: DependencyGraph;
  inferenceEngine: InferenceEngine;
}> = async (fastify, opts) => { ... };
```

For mapping internal `GraphDelta` to wire `GraphDeltaMessage`:
- `addedNodes` (string[] of node IDs) -> map each ID via `graph.getNodeMetadata(id)` to build `GraphNode` objects. Use the same mapping logic as `getSnapshot()` for name/nodeType/zone/fileList/edgeCounts.
- `modifiedNodes` (string[]) -> map to `updatedNodes: GraphNode[]` in the wire message
- `removedNodeIds`, `addedEdges`, `removedEdgeIds` map more directly

For `addedEdges` mapping: `GraphDeltaEdge` has `{ v, w, symbols }` but wire `GraphEdge` has `{ id, sourceId, targetId, edgeType }`. Map `v` -> `sourceId`, `w` -> `targetId`, generate `id` as `${v}->${w}`, edgeType defaults to `'imports_depends_on'`.

**2. Create `packages/server/src/plugins/snapshot.ts`:**

Simple REST endpoint:
```typescript
import type { FastifyPluginAsync } from 'fastify';
import type { DependencyGraph } from '../graph/DependencyGraph.js';
import type { InitialStateMessage } from '@archlens/shared/types';

export const snapshotPlugin: FastifyPluginAsync<{
  graph: DependencyGraph;
}> = async (fastify, { graph }) => {
  fastify.get('/api/snapshot', async (_req, reply) => {
    const { nodes, edges } = graph.getSnapshot();
    const response: InitialStateMessage = {
      type: 'initial_state',
      version: graph.getVersion(),
      nodes,
      edges,
      layoutPositions: {}, // Phase 6 will populate this from SQLite layout_positions table
    };
    return reply.send(response);
  });
};
```
