**3. Create `packages/client/src/store/graphStore.ts`:**

Build a Zustand store using `create<GraphStore>()((set, get) => ({...}))` (double-paren pattern for TypeScript middleware compat per RESEARCH.md).

Store shape:
```typescript
type ConnectionStatus = 'connecting' | 'connected' | 'disconnected' | 'syncing';

interface ChangeSummary {
  addedNodes: number;
  removedNodes: number;
  addedEdges: number;
  removedEdges: number;
}

interface GraphStore {
  nodes: Map<string, GraphNode>;
  edges: Map<string, GraphEdge>;
  version: number;
  connectionStatus: ConnectionStatus;
  changeSummary: ChangeSummary | null; // Set on reconnect when changes occurred while away

  // Actions — called by WsClient, not React components
  applyDelta: (msg: GraphDeltaMessage) => void;
  applySnapshot: (msg: InitialStateMessage) => void;
  setConnectionStatus: (status: ConnectionStatus) => void;
  setChangeSummary: (summary: ChangeSummary | null) => void;
}
```

`applyDelta` implementation:
- Create new Map copies from current state (immutable update for Zustand)
- Remove nodes by ID from `msg.removedNodeIds`
- Remove edges by ID from `msg.removedEdgeIds`
- Add/update nodes from `msg.addedNodes` and `msg.updatedNodes` (keyed by `node.id`)
- Add edges from `msg.addedEdges` (keyed by `edge.id`)
- Update version to `msg.version`

`applySnapshot` implementation:
- Build new Map from `msg.nodes` (keyed by `node.id`) and `msg.edges` (keyed by `edge.id`)
- Set version to `msg.version`
- Set connectionStatus to `'connected'`

Export both `useGraphStore` (React hook for Phase 6+) and `graphStore` (vanilla reference for WsClient — `useGraphStore` itself works as vanilla via `.getState()/.setState()`).

Import `GraphNode`, `GraphEdge`, `GraphDeltaMessage`, `InitialStateMessage` from `@archlens/shared/types`.

**1. Create `packages/client/src/ws/wsClient.ts`:**

Build a plain TypeScript class that manages the WebSocket lifecycle. This is intentionally hand-rolled (not a library) per RESEARCH.md recommendation because reconnecting-websocket is unmaintained.

Class structure:
```typescript
export class WsClient {
  private ws: WebSocket | null = null;
  private attempt: number = 0;
  private retryTimer: ReturnType<typeof setTimeout> | null = null;
  private pendingDeltas: GraphDeltaMessage[] = [];
  private batchTimer: ReturnType<typeof setTimeout> | null = null;
  private lastQueuedVersion: number = 0; // Tracks version of last queued delta (not store version)
  private previousVersion: number = 0; // Store version before disconnect — for change summary

  constructor() {}

  connect(): void { ... }
  destroy(): void { ... }
  private scheduleReconnect(): void { ... }
  private handleMessage(event: MessageEvent): void { ... }
  private queueDelta(msg: GraphDeltaMessage): void { ... }
  private flushDeltas(): void { ... }
  private requestSnapshot(): void { ... }
}
```

**Reconnect with exponential backoff:**
- Base delay: 500ms, max delay: 30,000ms (30 seconds)
- Formula: `Math.min(BASE_MS * 2^attempt + Math.random() * 1000, MAX_MS)` (jitter prevents thundering herd)
- Retry forever (per user decision: "Keep retrying forever with exponential backoff")
- Reset `attempt = 0` on successful open

**Connection lifecycle:**
- `onopen`: reset attempt counter, set connectionStatus to 'connected'. If `previousVersion > 0` and store version > previousVersion after snapshot, compute change summary ("+X nodes, +Y edges since last visit" per user decision). Reset `lastQueuedVersion` to store's current version.
- `onmessage`: parse via `ServerMessageSchema.safeParse(JSON.parse(event.data))`. If parse fails, `console.error('[WS] Invalid message:', parsed.error)` and return (silent drop per user decision). Route by `msg.type`:
  - `'initial_state'`: call `graphStore.getState().applySnapshot(msg)`. Update `lastQueuedVersion = msg.version`.
  - `'graph_delta'`: check version continuity against `lastQueuedVersion` (NOT store version — per RESEARCH.md Pitfall 3). If `msg.version !== lastQueuedVersion + 1`, trigger gap recovery. Otherwise `lastQueuedVersion = msg.version` and queue delta.
  - `'inference'`: store inference data. For Phase 5, inference messages are received but the store for inference results will be created in Phase 7. For now, log to console: `console.log('[WS] Inference:', msg)`. This is acceptable because the transport layer's job is to deliver messages — Phase 7 will add the inference store.
  - `'error'`: if `msg.code === 'DIRECTORY_UNAVAILABLE'`, set connectionStatus to a special state. For Phase 5, store a `lastError` field or just log: `console.warn('[WS] Server error:', msg.code, msg.message)`. The banner UI is Phase 7's concern.
- `onclose`: set connectionStatus to 'disconnected', save `previousVersion = graphStore.getState().version`, call `scheduleReconnect()`.
- `onerror`: no-op (onerror is always followed by onclose — let onclose handle reconnect per RESEARCH.md).

**500ms batch window (per user decision: "Batched updates ~500ms"):**
- `queueDelta(msg)`: push to `pendingDeltas`, start `batchTimer = setTimeout(flushDeltas, 500)` if not already running.
- `flushDeltas()`: drain `pendingDeltas`, apply each via `graphStore.getState().applyDelta(delta)` in order, clear timer.
- On version gap detection mid-batch: cancel `batchTimer`, clear `pendingDeltas`, request snapshot (per RESEARCH.md open question #4 recommendation).

**Version gap detection and snapshot recovery:**
- When `msg.version !== lastQueuedVersion + 1`:
  1. Set connectionStatus to 'syncing' (per user decision: "show brief 'Syncing...' indicator")
  2. Cancel any pending batch timer
  3. Clear pendingDeltas
  4. Call `requestSnapshot()` via REST `fetch('/api/snapshot')`
  5. On snapshot response: validate with `ServerMessageSchema.safeParse()`, apply via `applySnapshot()`, set connectionStatus to 'connected', update `lastQueuedVersion`

**IMPORTANT:** Initialize WsClient as a module-level singleton in `main.ts`, NOT inside a React component (per RESEARCH.md anti-pattern: "Using React useEffect for WebSocket initialization").

**2. Create `packages/client/src/utils/viewport.ts`:**

Simple localStorage read/write helpers for viewport persistence (per user decision: "Viewport restored from browser localStorage on reconnect"):

```typescript
const VIEWPORT_KEY = 'archlens:viewport';

export interface ViewportState {
  zoom: number;
  panX: number;
  panY: number;
}

export function saveViewport(state: ViewportState): void {
  try {
    localStorage.setItem(VIEWPORT_KEY, JSON.stringify(state));
  } catch {
    // localStorage quota or private browsing — silently ignore
  }
}

export function loadViewport(): ViewportState | null {
  try {
    const raw = localStorage.getItem(VIEWPORT_KEY);
    if (!raw) return null;
    const parsed = JSON.parse(raw);
    if (typeof parsed.zoom === 'number' && typeof parsed.panX === 'number' && typeof parsed.panY === 'number') {
      return parsed as ViewportState;
    }
    return null;
  } catch {
    return null;
  }
}
```

Phase 6 (canvas renderer) will call `saveViewport()` on every viewport change and `loadViewport()` on mount. Phase 5 just provides the helpers.
