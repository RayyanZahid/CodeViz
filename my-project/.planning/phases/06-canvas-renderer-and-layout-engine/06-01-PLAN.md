  <action>
1. **Rename main.ts to main.tsx** — delete main.ts, create main.tsx with:
   - Import `{ createRoot }` from `react-dom/client`
   - Import `{ WsClient }` from `./ws/wsClient.js` (keep module-level singleton pattern from Phase 5)
   - Import `{ App }` from `./App.js`
   - Initialize WsClient singleton and call connect() BEFORE React mount (Phase 5 pattern preserved)
   - Call `createRoot(document.getElementById('app')!).render(<App />)` — no StrictMode wrapper (avoids double-mount Konva issues per RESEARCH.md Pitfall 1)

2. **Create App.tsx** — minimal root component:
   - Use `useEffect` + `useRef` to track container div dimensions (width/height)
   - Render a full-viewport div (100vw x 100vh, overflow hidden, dark background e.g. `#0a0a0f`)
   - Pass width/height to `<ArchCanvas width={width} height={height} />`
   - Add a `useEffect` with `ResizeObserver` on the container to update width/height on window resize

3. **Create canvas/ArchCanvas.tsx** — the two-layer Konva stage:
   - Import `{ Stage, Layer }` from `react-konva`
   - Import `type Konva` from `konva`
   - Import `{ graphStore }` from `../store/graphStore.js`
   - Declare refs: `graphLayerRef = useRef<Konva.Layer>(null)`, `animLayerRef = useRef<Konva.Layer>(null)`, `stageRef` for Stage
   - In useEffect:
     - Guard: if graphLayer or animLayer null, return
     - Set up imperative graphStore.subscribe() — for now, just log delta to console (Plans 02/03 will add NodeRenderer/EdgeRenderer/layout)
     - Return unsub cleanup function
   - Render:
     ```tsx
     <Stage ref={stageRef} width={width} height={height} draggable>
       <Layer ref={graphLayerRef} id="graph-layer" />
       <Layer ref={animLayerRef} id="anim-layer" listening={false} />
     </Stage>
     ```
   - Stage draggable={true} enables pan (REND-02 foundation)
   - Animation layer has listening={false} (no hit detection needed — Pitfall 2)
   - Graph layer keeps listening enabled for click-to-select (CONTEXT.md decision)

4. **Create layout/ZoneConfig.ts** — zone geometry constants:
   - Define a `CANVAS_WIDTH = 2400` and `CANVAS_HEIGHT = 1600` for the virtual canvas space
   - Define `ZoneBounds` interface: `{ x0: number; y0: number; x1: number; y1: number }`
   - Define `ZoneLayout` interface: `{ name: string; label: string; bounds: ZoneBounds; center: { x: number; y: number }; fillColor: string; bgColor: string; glowColor: string }`
   - Create `ZONE_LAYOUTS` array with 6 entries matching CONTEXT.md zone positions:
     - **frontend**: left region (~x: 0-380), blue-tinted fill (e.g. `#2563eb` fill, `#2563eb10` bg, `#60a5fa` glow)
     - **api**: center-left (~x: 400-780), green-tinted (e.g. `#16a34a` fill, `#16a34a10` bg, `#4ade80` glow)
     - **services**: center (~x: 800-1580), purple-tinted (e.g. `#9333ea` fill, `#9333ea10` bg, `#c084fc` glow)
     - **data-stores**: right (~x: 1600-2000), amber-tinted (e.g. `#d97706` fill, `#d9770610` bg, `#fbbf24` glow)
     - **infrastructure**: bottom strip (~y: 1200-1560), slate-tinted (e.g. `#475569` fill, `#47556910` bg, `#94a3b8` glow)
     - **external**: outer ring / top-right (~x: 2020-2380), gray-tinted (e.g. `#6b7280` fill, `#6b728010` bg, `#9ca3af` glow)
   - Export `ZONE_LAYOUTS`, `CANVAS_WIDTH`, `CANVAS_HEIGHT`
   - Export helper: `getZoneLayout(zoneName: string): ZoneLayout` — returns matching zone or falls back to 'external' zone for 'unknown' or unrecognized names
   - Export helper: `getZoneCenter(zoneName: string): { x: number; y: number }` — shortcut for placement
   - Centers computed as midpoint of bounds

NOTE: Do NOT use React StrictMode (`<StrictMode>`) wrapper — per RESEARCH.md Pitfall 1, double-mount breaks Konva imperative subscriptions.
