Create `layout/IncrementalPlacer.ts` implementing the sticky incremental layout engine:

1. **Imports:**
   - `{ forceSimulation, forceLink, forceManyBody, forceX, forceY, forceCollide }` from `d3-force`
   - `{ forceBoundary }` from `d3-force-boundary`
   - `{ getZoneLayout, getZoneCenter, ZONE_LAYOUTS, CANVAS_WIDTH, CANVAS_HEIGHT }` from `./ZoneConfig.js`
   - `type { GraphNode, GraphEdge }` from `@archlens/shared/types`

2. **SimNode interface** (extends d3-force's node interface):
   ```typescript
   interface SimNode {
     id: string;
     x: number;
     y: number;
     vx?: number;
     vy?: number;
     fx?: number | null;
     fy?: number | null;
     zone: string;
     index?: number;
   }
   ```
   CRITICAL: Do NOT pass GraphNode objects from the store directly to d3-force — it mutates them in-place (RESEARCH.md Pitfall 3). Always create fresh SimNode objects.

3. **Class IncrementalPlacer:**

   - Private `positions: Map<string, { x: number; y: number }>` — the canonical position store
   - Private `simulation: d3.Simulation<SimNode, undefined>` — created in constructor, stopped immediately

   - Constructor:
     - Create simulation with:
       - `forceManyBody().strength(-30)` — repulsion between nodes
       - `forceCollide().radius(40)` — prevent node overlap
       - `alphaDecay(0.05)` — fast convergence for incremental updates
     - `simulation.stop()` — NEVER run animated; always manual tick (RESEARCH.md anti-pattern)

   - Method `loadPositions(layoutPositions: Record<string, { x: number; y: number; zone: string | null }>)`:
     - For each entry in layoutPositions (from InitialStateMessage.layoutPositions):
       - Store in positions map
     - Called once on snapshot load — these become sticky immediately

   - Method `placeNewNodes(allNodes: Map<string, GraphNode>, edges: Map<string, GraphEdge>)`:
     - Build simNodes array: for each node in allNodes:
       - If positions has this node (existing): create SimNode with fx/fy set to stored position (PINNED — they will NOT move)
       - If positions does not have this node (new): create SimNode with x/y initialized to zone center (no fx/fy — FREE to move)
     - If no new nodes exist (all have positions), return early — nothing to do (LAYOUT-04: no reshuffle)
     - Build sim links: for each edge, create `{ source: edge.sourceId, target: edge.targetId }`
     - Update simulation:
       - `simulation.nodes(simNodes)`
       - Set `force('link', forceLink(simLinks).id((d: SimNode) => d.id).distance(80).strength(0.3))`
       - Set zone centering forces:
         - `force('x', forceX<SimNode>((d) => getZoneCenter(d.zone).x).strength(0.3))`
         - `force('y', forceY<SimNode>((d) => getZoneCenter(d.zone).y).strength(0.3))`
       - Set zone boundary forces:
         - For EACH zone in ZONE_LAYOUTS, add a boundary force that only applies to nodes in that zone. Since d3-force-boundary applies to ALL nodes, implement zone containment by using a single `forceBoundary` with per-node strength:
           - Use `forceBoundary(0, 0, CANVAS_WIDTH, CANVAS_HEIGHT).strength(0.5)` as a global boundary
           - The forceX/forceY with zone centers handles zone containment (soft constraint)
           - Alternatively: apply post-tick clamping as a fallback: after all ticks, clamp each new node's position to its zone bounds
     - Run simulation: `simulation.alpha(0.3).restart()` then `simulation.stop()`, then `simulation.tick(50)` — 50 ticks is enough for new nodes to settle near zone centers
     - After simulation completes:
       - For each simNode that was NEW (no prior position):
         - Clamp to zone bounds as safety net
         - Store position: `positions.set(id, { x: node.x, y: node.y })`
       - Existing nodes: their fx/fy prevented any movement; no position update needed
     - Return the Map of ALL positions (existing + newly placed)

   - Method `getPositions(): Map<string, { x: number; y: number }>`:
     - Return readonly reference to positions map

   - Method `getPosition(nodeId: string): { x: number; y: number } | undefined`:
     - Return from positions map

   - Method `removeNode(nodeId: string)`:
     - Delete from positions map (called when node is removed from graph)

   - Method `hasPosition(nodeId: string): boolean`:
     - Check if positions map has this node

   - Method `getNewNodeCount(allNodes: Map<string, GraphNode>): number`:
     - Count nodes in allNodes that are NOT in positions map
     - Used for early-exit optimization

4. **Post-tick zone clamping helper** (private method):
   ```typescript
   private clampToZone(x: number, y: number, zoneName: string): { x: number; y: number } {
     const zone = getZoneLayout(zoneName);
     const bounds = zone.bounds;
     const padding = 20; // Keep nodes away from zone edges
     return {
       x: Math.max(bounds.x0 + padding, Math.min(bounds.x1 - padding, x)),
       y: Math.max(bounds.y0 + padding, Math.min(bounds.y1 - padding, y)),
     };
   }
   ```

IMPORTANT per RESEARCH.md:
- NEVER call `simulation.restart()` in animated mode — always `stop()` then `tick(N)` for silent computation
- ALWAYS set `fx`/`fy` on placed nodes to prevent drift on subsequent runs
- ALWAYS create fresh SimNode objects — do not pass store GraphNode objects to d3-force
