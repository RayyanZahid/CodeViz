<objective>
Fix Zod schema gaps so component data fields flow through validation, and expose a reusable file-to-component ID mapping from ComponentAggregator.

Purpose: Client-side Zod validation currently strips `fileCount`, `keyExports` (on nodes) and `dependencyCount` (on edges) because these fields are absent from the schemas. The ComponentAggregator already populates these fields, but they are silently dropped at the client boundary. Additionally, Plan 02 needs a file-to-component lookup map for translating inference IDs.

Output: Updated Zod schemas that pass all component fields through to the client stores, and a `getFileToComponentMap()` method on ComponentAggregator.
</objective>
