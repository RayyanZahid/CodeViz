## Implementation Decisions

### Build command scope
- `build` script at root runs `tsc -b` across all packages (shared, server, client) — TypeScript only, no Vite production bundle
- Uses pnpm internally (e.g., `pnpm -r build` or direct `tsc -b` with project references)
- Keep `typecheck` script separate from `build` — both exist independently
- All three packages (shared, server, client) are included in the build

### Test vs code alignment
- Journey test should use `pnpm build` instead of `npm run build` — match the actual project tooling
- Keep the 2-second wait + zero-JS-errors check as-is
- Test should verify build outputs exist: check that dist folders contain `.js` files after build

### Dev server strategy
- Add `webServer` block to Playwright config to auto-start the dev server
- Start full stack via root `pnpm dev` (both Vite client and Fastify server)
- 60-second timeout for server readiness (accounts for first-run compilation)
- Playwright manages server lifecycle (start before tests, stop after)
