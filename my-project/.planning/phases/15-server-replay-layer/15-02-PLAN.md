Create `packages/server/src/snapshot/IntentAnalyzer.ts` — a new class parallel to `SnapshotManager` in the `snapshot/` directory.

**Class design:**

```typescript
import type { DependencyGraph } from '../graph/DependencyGraph.js';
import type { GraphDelta } from '@archlens/shared/types';
import { IntentCategory } from '@archlens/shared/types';
import type { IntentSession } from '@archlens/shared/types';
import { intentSessionsRepository } from '../db/repository/intentSessions.js';
import { snapshotsRepository } from '../db/repository/snapshots.js';
import { broadcast } from '../plugins/websocket.js';
```

**Constructor:** Receives `graph: DependencyGraph`, `sessionId: string`, `watchRoot: string`. Subscribes to `graph.on('delta', ...)` using a bound handler (same pattern as SnapshotManager). Has `destroy()` to remove the listener.

**Core algorithm — additive scoring with EWMA decay:**

1. Maintain a `scores` record: `Record<string, number>` initialized to 0 for each IntentCategory value.

2. On each delta:
   a. Decay all existing scores by multiplying by `DECAY = 0.85`
   b. Run `classifyDelta(delta)` to produce signal array
   c. Add signal weights to respective category scores
   d. Compute winner = category with highest score
   e. Compute confidence = topScore / sumOfAllScores (0-1 float). If sum is 0, confidence = 0.
   f. Call `evaluateSession(winner, confidence)`

3. `classifyDelta(delta: GraphDelta)` returns `Array<{category: IntentCategory, weight: number}>`:

   **File-path signals** (from `[...delta.addedNodes, ...delta.modifiedNodes, ...delta.triggerFiles]`):
   - File contains `test`, `spec`, or `__tests__` -> TEST_WRITING weight 0.6
   - File contains `package.json`, `package-lock`, or `pnpm-lock` -> DEPENDENCY_UPDATE weight 0.8
   - File contains `.config.`, `tsconfig`, `vite.config`, or `drizzle` -> CLEANUP weight 0.3 (config changes often accompany cleanup)

   **Topology signals** (from delta shape):
   - `addedNodes.length > 0` OR `addedEdges.length > 1` -> FEATURE_BUILDING weight 0.5
   - `removedNodeIds.length > 0` OR `removedEdgeIds.length > 1` -> REFACTORING weight 0.4
   - `cyclesAdded.length > 0` OR `cyclesRemoved.length > 0` -> CLEANUP weight 0.5
   - Pure modification (modifiedNodes > 0, no adds/removes of nodes or edges) -> BUG_FIXING 0.3, REFACTORING 0.2

4. **Session lifecycle (`evaluateSession`):**
   - If no active session: call `openSession(category, confidence)`
   - If category differs from active session's category: close current session (`closeSession`), open new one (`openSession`) -- this is the focus shift per CONTEXT.md
   - If same category: call `updateSession(confidence)`

5. **`openSession(category, confidence)`:**
   - Look up `startSnapshotId` via `snapshotsRepository.getLatestId(this.sessionId)` — may be undefined (null) if no snapshots written yet (per Pitfall 3 in RESEARCH.md)
   - Map category to human-readable objective string: FEATURE_BUILDING -> "Adding Feature", BUG_FIXING -> "Bug Fix", REFACTORING -> "Refactoring", TEST_WRITING -> "Test Coverage", DEPENDENCY_UPDATE -> "Dependency Update", CLEANUP -> "Cleanup"
   - Insert via `intentSessionsRepository.insert(...)` with sessionId, watchRoot, category, objective, confidence, subtasks: [], evidence: [], riskSnapshot: [], startSnapshotId (or null), endSnapshotId: null, startedAt: new Date()
   - Store the returned ID as `this.activeSessionId` and the category as `this.activeCategory`
   - Build an IntentSession object and broadcast `{ type: 'intent_updated', session: ... }` via `broadcast()`

6. **`closeSession()`:**
   - Look up `endSnapshotId` via `snapshotsRepository.getLatestId(this.sessionId)`
   - Call `intentSessionsRepository.close(this.activeSessionId, endSnapshotId ?? null, new Date())`
   - Broadcast `{ type: 'intent_closed', sessionId: this.sessionId, endSnapshotId: endSnapshotId ?? null }`
   - Clear `this.activeSessionId` and `this.activeCategory`

7. **`updateSession(confidence)`:**
   - Call `intentSessionsRepository.updateConfidence(this.activeSessionId, confidence, this.activeObjective)`
   - Only broadcast `intent_updated` if confidence changed by more than 0.05 from last broadcast value (avoid flooding WebSocket)

8. **Activity gap handling (per STATE.md blocker note):**
   - Track `lastDeltaTimestamp`. If gap > 90_000ms (90 seconds), reset all scores to 0 before processing the new delta — treat as a fresh context. This prevents stale scores from creating spurious classifications.

9. **Minimum signal threshold:**
   - After computing winner, only emit/update session if the sum of all scores > 0.1. This prevents noise from triggering spurious focus shifts on very low activity.

10. **`destroy()`:**
    - Remove delta listener from graph
    - If active session exists, close it (so it doesn't dangle as open in the DB)

**3. Modify `captureSnapshot()` method:**

After the `snapshotsRepository.insert(...)` call (step 6) and BEFORE the existing FIFO pruning (step 7), add checkpoint creation logic:

```typescript
// 6b. Checkpoint creation — every CHECKPOINT_INTERVAL snapshots
if (this.sequenceNumber % CHECKPOINT_INTERVAL === 0) {
  checkpointsRepository.insert({
    sessionId: this.sessionId,
    watchRoot: this.watchRoot,
    sequenceNumber: this.sequenceNumber,
    snapshotId: insertedId,
    createdAt: new Date(),
  });
  // Prune old checkpoints beyond MAX_CHECKPOINTS
  const cpCount = checkpointsRepository.getCount(this.sessionId);
  if (cpCount > MAX_CHECKPOINTS) {
    checkpointsRepository.deleteOldest(this.sessionId);
  }
  console.log(`[Snapshot] Checkpoint created at sequence #${this.sequenceNumber}`);
}
```

**4. Replace FIFO pruning** (step 7) to use `deleteOldestNonCheckpoint` instead of `deleteOldest`:

Change:
```typescript
// 7. FIFO pruning — remove oldest if over cap
const count = snapshotsRepository.getCount(this.sessionId);
if (count > MAX_SNAPSHOTS) {
  snapshotsRepository.deleteOldest(this.sessionId);
}
```

To:
```typescript
// 7. FIFO pruning — remove oldest NON-CHECKPOINT if over cap
const count = snapshotsRepository.getCount(this.sessionId);
if (count > MAX_SNAPSHOTS) {
  snapshotsRepository.deleteOldestNonCheckpoint(this.sessionId);
}
```

This ensures checkpoint snapshots are never deleted by the FIFO pruner — they are only removed by the checkpoint pruner (which enforces MAX_CHECKPOINTS). Per RESEARCH.md Pitfall 2.

**IMPORTANT:** The existing `deleteOldest` method should NOT be removed from the repository — it may still be used elsewhere. Only the call site in SnapshotManager changes.
