## Common Pitfalls

### Pitfall 1: Wrong Git Path Prefix

**What goes wrong:** `git checkout HEAD -- .auto-gsd/journey-tests/...` fails with "pathspec did not match any files".
**Why it happens:** The git repo root is at `F:\Directory\FrontierIntelligenceHack` (one level above the working directory `my-project/`). The correct path inside git is `my-project/.auto-gsd/journey-tests/...`.
**How to avoid:** Always prefix paths with `my-project/` in git commands, or use `git restore --worktree --source=HEAD` from the correct relative context.
**Warning signs:** `git show HEAD:.auto-gsd/journey-tests/...` returns an error mentioning a `my-project/` hint.

Verified: `git show "HEAD:my-project/.auto-gsd/journey-tests/journey-build-and-start.spec.ts"` works correctly.

### Pitfall 2: Staged vs Unstaged — Both Must Be Reverted

**What goes wrong:** Only unstaged changes are reverted, leaving staged changes with placeholder content.
**Why it happens:** `git diff HEAD` shows both staged (`M`) and unstaged (` M`) modifications. The git status shows all 4 files are modified.
**How to avoid:** `git checkout HEAD -- <path>` handles both staged and unstaged simultaneously by resetting the index AND working tree to HEAD. No separate `git reset HEAD <path>` needed.
**Warning signs:** After restore, `git status` still shows the file modified.

### Pitfall 3: Playwright Cannot Find Dev Server

**What goes wrong:** Tests fail immediately with "page.goto: net::ERR_CONNECTION_REFUSED" because webServer wasn't started.
**Why it happens:** If the current `playwright.config.ts` (placeholder) is used instead of the restored version, the `webServer` block is absent and Playwright won't auto-start the server.
**How to avoid:** Restore `playwright.config.ts` first, before running the test suite.
**Warning signs:** The placeholder `playwright.config.ts` has no `webServer` block and no `workers: 1`.

### Pitfall 4: Test Timeout on Cold Build

**What goes wrong:** "App builds without errors" test times out.
**Why it happens:** The committed `journey-build-and-start.spec.ts` uses `timeout: 180000` for the `pnpm build` execSync call. If the timeout was reduced to 30000 (placeholder value) a full TypeScript monorepo compilation would time out.
**How to avoid:** Ensure `journey-build-and-start.spec.ts` is restored to the HEAD version with the 180000ms timeout.
**Warning signs:** Build test times out at exactly 30 seconds.

### Pitfall 5: Server Must Have NODE_ENV=development for Diagnostic Endpoints

**What goes wrong:** Phase 14 tests fail with 404 on `/api/debug/*` endpoints.
**Why it happens:** The `webServer.env.NODE_ENV: 'development'` config in `playwright.config.ts` enables the diagnostic plugin. Without it, the server starts in production mode and diagnostic endpoints are unavailable.
**How to avoid:** The restored `playwright.config.ts` includes `env: { NODE_ENV: 'development' }` in the webServer block.
**Warning signs:** GET `/api/debug/graph-snapshots-table` returns 404 when server is running.
