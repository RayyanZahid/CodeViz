## Root Cause Documentation

**This is the 4th occurrence** (15.1, 15.2, 15.4, 15.5) of auto-gsd overwriting real journey test files with placeholder stubs. The root cause is definitively identified:

**Source:** `OttoGSD/src/orchestrator/journey-test-generator.ts`

**Trigger:** `phase:complete` event in `journey-orchestrator.ts` fires `generateTestScripts()` after EVERY phase completion.

**Mechanism:**
1. Reads `JOURNEYS.md` to derive `JourneyDefinition[]` with `text_contains` type assertions
2. **Deletes ALL files starting with `journey-` in `.auto-gsd/journey-tests/`**
3. Regenerates them as placeholder Playwright stubs via `generatePlaywrightSpec()` — only body-length checks (`bodyText.length > 0`), using `BASE_URL` (Vite 5173) instead of `SERVER_URL` (Fastify 3100)
4. Also regenerates `playwright.config.ts` with a minimal config that lacks `workers: 1`, `webServer`, and `NODE_ENV: development`

**Stub config hardcoded in `generatePlaywrightConfig()`:**
```typescript
function generatePlaywrightConfig(): string {
  return `import { defineConfig } from '@playwright/test';
export default defineConfig({
  testDir: '.',
  timeout: 30000,
  retries: 0,
  use: {
    baseURL: process.env.BASE_URL || 'http://localhost:5173',
    headless: true,
  },
  reporter: [['json', { outputFile: 'results.json' }]],
});`;
}
```

**No "skip if file already has real content" guard exists in the generator.**

**Prevention requires OttoGSD binary changes** — out of scope per CONTEXT.md decision and Phase 15.4 STATE.md decision.

**Standard mitigation (restore-from-HEAD playbook):**
```bash
cd /f/Directory/FrontierIntelligenceHack/my-project

git checkout HEAD -- .auto-gsd/journey-tests/playwright.config.ts
```
