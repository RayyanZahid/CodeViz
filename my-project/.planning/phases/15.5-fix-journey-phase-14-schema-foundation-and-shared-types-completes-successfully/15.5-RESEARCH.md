## Summary

The 4 journey test files were overwritten by the OttoGSD auto-gsd orchestrator's `journey-test-generator.ts` module. This TypeScript module runs after EVERY phase completion, reads the JOURNEYS.md assertions, and regenerates ALL `journey-*.spec.ts` files AND `playwright.config.ts` from scratch using placeholder stub templates. The stubs use `text_contains` assertions that merely check `bodyText.length > 0`, replacing the real deep assertions that hit diagnostic endpoints.

The good news is that git HEAD already has the correct versions of all 4 files — the uncommitted working tree contains the stubs. The CONTEXT.md decision to use `git checkout d39ce2e -- <files>` is correct but note that HEAD already matches d39ce2e for these files (no commits touched them after d39ce2e). So `git checkout HEAD -- <files>` is sufficient and equivalent.

The root cause is unambiguous: `OttoGSD/src/orchestrator/journey-test-generator.ts` deletes all files starting with `journey-` in `.auto-gsd/journey-tests/` and regenerates them from the structured `JourneyDefinition[]` derived from JOURNEYS.md success criteria. It also regenerates `playwright.config.ts` with a minimal config that lacks `workers: 1`, `webServer`, and `NODE_ENV: development`. There is no "skip if file exists" logic. Prevention within project config/settings is not directly possible because the generator runs in the auto-gsd TypeScript runtime (OttoGSD), not in the GSD Claude commands.

**Primary recommendation:** Restore all 4 files from HEAD using `git -C /f/Directory/FrontierIntelligenceHack/my-project checkout HEAD -- my-project/.auto-gsd/journey-tests/journey-build-and-start.spec.ts my-project/.auto-gsd/journey-tests/journey-phase-14.spec.ts my-project/.auto-gsd/journey-tests/journey-phase-15.spec.ts my-project/.auto-gsd/journey-tests/playwright.config.ts`, verify 12/12 tests pass, then commit the restored files to create a new git HEAD that is ahead of the auto-gsd-triggered overwrite and document the root cause.

### Anti-Patterns to Avoid
- **Using `npm run build` in build-and-start test:** The correct command is `pnpm build` (project uses pnpm workspaces). The HEAD version of journey-build-and-start.spec.ts already uses `pnpm build` with `cwd: projectRoot`.
- **Parallel test execution:** `workers: 1` is mandatory — tests share server state via `POST /api/watch`. Parallel execution causes race conditions.
- **Missing `NODE_ENV=development`:** Diagnostic endpoints are only registered when `NODE_ENV=development`. The real playwright.config.ts sets this in `webServer.env`.
- **Using BASE_URL (5173) for API tests:** Phase 14 and 15 tests use `SERVER_URL` (port 3100). Only build-and-start tests use `BASE_URL` (port 5173).

## Common Pitfalls

### Pitfall 1: Wrong Git Path Syntax
**What goes wrong:** `git checkout HEAD -- .auto-gsd/journey-tests/...` fails because the working directory's git root is the parent of `my-project/`.
**Why it happens:** The git repository is rooted at `/f/Directory/FrontierIntelligenceHack`, not at `/f/Directory/FrontierIntelligenceHack/my-project`. Git commands run from inside `my-project/` work correctly because git resolves paths relative to the repo root automatically.
**How to avoid:** Run `git checkout HEAD -- .auto-gsd/...` from within the `my-project` directory — git will handle the path correctly.
**Warning signs:** Error message `fatal: pathspec '...' did not match any file(s) known to git`

### Pitfall 2: Stale Playwright Node_Modules
**What goes wrong:** `npx playwright test` fails with "cannot find @playwright/test" or browser not installed.
**Why it happens:** The `.auto-gsd/journey-tests/node_modules` may be missing or stale after the auto-gsd run.
**How to avoid:** Run `npm install` in `.auto-gsd/journey-tests/` before running tests. The package.json is standalone (not in pnpm workspace) so `npm install` (not `pnpm install`) works.
**Warning signs:** Error `Cannot find module '@playwright/test'`

### Pitfall 3: Server Not Running at Port 3100
**What goes wrong:** Phase 14 tests fail with connection refused on port 3100.
**Why it happens:** The Vite dev server starts on 5173, but the Fastify API server runs on 3100. The playwright.config.ts `webServer` only starts the Vite server. Phase 14/15 tests access port 3100 directly via `SERVER_URL`.
**How to avoid:** The full stack starts via `pnpm dev` which starts both servers. Confirm port 3100 is live with `curl http://localhost:3100/api/debug/shared-types` before running Playwright.
**Warning signs:** Phase 14/15 tests timeout or get ECONNREFUSED; build-and-start tests pass but phase-14/15 fail

### Pitfall 4: Committed Stubs Mistaken for Real Files
**What goes wrong:** Developer assumes working tree files are correct and doesn't restore from git.
**Why it happens:** The working tree currently has stubs (overwritten by auto-gsd). However git HEAD (commit 2e390e5) still has the correct files — the overwrite is only in the working tree, not committed.
**How to avoid:** Always `git diff HEAD -- .auto-gsd/journey-tests/journey-phase-14.spec.ts` first to confirm whether restoration is needed.
**Warning signs:** Tests pass with minimal `bodyText.length > 0` checks — this means stubs are running, not real assertions.

### Pitfall 5: Root Cause Prevention Is Outside This Codebase
**What goes wrong:** Attempting to configure GSD settings to prevent the overwrite, spending time searching through .planning/config.json or workflow files.
**Why it happens:** The overwrite logic is in `OttoGSD/src/orchestrator/journey-test-generator.ts` (an external binary), not in the GSD Claude command system. The journey test generation runs unconditionally after each phase.
**How to avoid:** Accept this per STATE.md Phase 15.4 decision: "No regression prevention mechanism added — accept the auto-GSD overwrite pattern and follow the same restore-from-HEAD playbook each time". Document the root cause but do not attempt to fix it in this phase.
**Warning signs:** Time spent in .claude/ workflow files searching for a config flag that would prevent overwriting.

**Root Cause Summary (definitive findings):**

The overwrite source is `OttoGSD/src/orchestrator/journey-test-generator.ts`. This module:
1. Runs after EVERY phase completes (triggered by `phase:complete` event in `journey-orchestrator.ts`)
2. Reads JOURNEYS.md to derive `JourneyDefinition[]` with `text_contains` type assertions
3. Deletes ALL files starting with `journey-` in `.auto-gsd/journey-tests/`
4. Regenerates them as placeholder Playwright stubs using `generatePlaywrightSpec()` which generates only body-length checks for `text_contains` assertions
5. Also regenerates `playwright.config.ts` with a minimal config that lacks `workers: 1`, `webServer`, and `NODE_ENV: development`

The `generatePlaywrightConfig()` function in that module hardcodes a stub config:
```typescript
// Source: OttoGSD/src/orchestrator/journey-test-generator.ts
function generatePlaywrightConfig(): string {
  return `import { defineConfig } from '@playwright/test';
export default defineConfig({
  testDir: '.',
  timeout: 30000,
  retries: 0,
  use: {
    baseURL: process.env.BASE_URL || 'http://localhost:5173',
    headless: true,
  },
  reporter: [['json', { outputFile: 'results.json' }]],
});`;
}
```

There is no "skip if file already has real content" guard in the generator. Prevention requires changes to the OttoGSD binary (out of scope per CONTEXT.md decision).
