**epochDetection.ts** — Pure utility module for epoch detection.

Export an `EpochMarker` interface:
```typescript
export interface EpochMarker {
  snapshotId: number;
  snapshotIndex: number;
  label: string;
  type: 'focus_shift' | 'gap';
  timestamp: number;
}
```

Export `detectEpochs(snapshots: SnapshotMeta[], intentSessions: IntentSession[]): EpochMarker[]`:
- From intent session boundaries: iterate `intentSessions` (sorted by startedAt). For consecutive sessions where prev.category !== curr.category, find the closest snapshot index to the transition timestamp (prev.endedAt ?? curr.startedAt). Create epoch with label `"{prevCategory} → {currCategory}"` (categories formatted: split on '_', capitalize), type: 'focus_shift'.
- From activity gaps: iterate snapshots. If gap between `snapshots[i].timestamp - snapshots[i-1].timestamp > 90_000` (90 seconds), create epoch with label "Activity resumed", type: 'gap'.
- Return sorted by timestamp ascending.
- Export a `findClosestSnapshotIndex(snapshots: SnapshotMeta[], timestamp: number): number` helper used by detectEpochs.

**TimelineBar.tsx** — Full-width 60px timeline strip.

Component props: `{ onExitReplay: () => void }`

**On mount (`useEffect([], [])`):**
- Fetch `GET /api/timeline` and `GET /api/intents` in parallel via `Promise.all`
- Call `replayStore.getState().setSnapshots(metas)` with timeline response
- Call `intentStore.getState().loadHistory(sessions)` with intents response
- Store the intent sessions in a local ref for epoch detection

**Layout structure (60px height, horizontal flex):**
```
[ PlaybackControls (left, ~120px) ] [ Timeline Track (flex: 1, relative) ]
```

**PlaybackControls sub-component (left section):**
- Play/pause button: shows ▶ when paused, ❚❚ when playing. Clicking toggles `replayStore.setIsPlaying`.
  - When snapshots.length === 0, disable (grey out).
  - If not in replay when clicked and snapshots exist, enter replay at the first snapshot.
- Step forward button: ▶| icon. Increments currentSnapshotIndex by 1, calls `loadSnapshotAndEnterReplay(snapshots[newIndex].id)`. Disabled when at last snapshot or snapshots empty.
- Step backward button: |◀ icon. Decrements currentSnapshotIndex by 1, calls `loadSnapshotAndEnterReplay`. Disabled when currentSnapshotIndex <= 0.
- Speed selector: clickable text showing current speed ("1x"). Clicking cycles: 1 → 2 → 4 → 0.5 → 1. Calls `replayStore.setPlaybackSpeed`.

**Timeline Track (center section, position: relative):**

1. **Heatmap background:** A row of `div` elements (one per snapshot, or grouped into ~100 buckets if many snapshots). Each div's opacity represents snapshot density in that time bucket. Opacity range: 0.1 (sparse) to 0.6 (dense). Background color: `rgba(100, 160, 255, opacity)`. This creates a subtle blue activity heatmap.

2. **Epoch tick marks:** For each epoch from `detectEpochs()` (memoized via useMemo on snapshots + intentHistory):
   - A vertical line (2px wide, height 100%) at the epoch's proportional x position
   - Color: `rgba(255, 255, 255, 0.4)` for gap epochs, `#eab308` for focus_shift epochs
   - Short text label below/above the tick (truncated to ~15 chars)
   - `title` attribute with full epoch label for hover tooltip
   - onClick: jump to that epoch's snapshot via `loadSnapshotAndEnterReplay`

3. **Timestamp axis labels:** At regular intervals along the track (every ~25% of width), show absolute time labels formatted with `Intl.DateTimeFormat(undefined, { hour: 'numeric', minute: '2-digit' })` — produces "2:15 PM" format (same as ReplayBanner.tsx). Only show if snapshots.length > 0.

4. **Draggable thumb:** A circular div (14px diameter, white border, blue fill) positioned at `(currentSnapshotIndex / (snapshots.length - 1)) * trackWidth`. Only visible when in replay mode or when dragging.
   - Use pointer capture pattern for drag: `onPointerDown` → `setPointerCapture`, `onPointerMove` → update visual position only (NO snapshot fetch during drag), `onPointerUp` → `releasePointerCapture` + compute final snapshot index + call `loadSnapshotAndEnterReplay`.
   - During drag, show a tooltip above the thumb with relative time ("3 minutes ago") using the timestamp of the nearest snapshot.

5. **Live-edge pulsing dot:** A 10px green (#22c55e) circle at the rightmost position. Pulse animation via CSS keyframes (opacity oscillation 0.4 to 1.0, 2s period). Only visible when snapshots.length > 0 and NOT at the live edge during replay.

6. **Click-to-enter-replay:** Clicking anywhere on the track (not on the thumb) computes the nearest snapshot index from click x-position and calls `loadSnapshotAndEnterReplay`. Set `replayStore.setCurrentSnapshotIndex`.

7. **Shift-click for diff:** If already in replay and shift key is held during click, instead of navigating, set `replayStore.setDiffBase(clickedSnapshotId)`. Show a second marker (amber diamond) at the diff base position.

**State reads from stores:**
- `useReplayStore(s => s.snapshots)` — snapshot metadata array
- `useReplayStore(s => s.currentSnapshotIndex)` — current position
- `useReplayStore(s => s.isReplay)` — mode gate
- `useReplayStore(s => s.isPlaying)` — playback state
- `useReplayStore(s => s.playbackSpeed)` — speed multiplier
- `useReplayStore(s => s.diffBaseSnapshotId)` — diff base
- `useIntentStore(s => s.intentHistory)` — for epoch detection

**Styling:** Dark background (`#0d0d14`), top border (`1px solid rgba(255,255,255,0.08)`), monospace fonts, inline styles (project convention). Track area has left padding to align with canvas edge.

**CSS animation for live-edge pulse:** Inject `@keyframes` via `useEffect` that creates a `<style>` element in `document.head` (same pattern as ReplayBanner.tsx pulse animation). Clean up on unmount.

Import `loadSnapshotAndEnterReplay` from `../canvas/ArchCanvas.js`.
Import `SnapshotMeta, IntentSession` from `@archlens/shared/types`.
